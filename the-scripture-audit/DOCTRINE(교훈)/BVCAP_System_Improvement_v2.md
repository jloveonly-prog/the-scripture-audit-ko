## 2. 신규 TYPE 제안

### TYPE-B-ψ: 심리적 시간차 (Psychological Temporal Differential)

```
발동 조건: 동일 인물이 동일 사건에 대해 시간 경과 후 행동이 달라진 경우
원리:      충격/감정 상태가 인물의 행동·발언을 결정하며,
           시간 경과 후 감정이 변화하면 행동도 변화함
```

**적용 사례:**
- 막 16:8 (충격·도주·침묵) → 마 28:8 (진정·기쁨·전달) = 동일 여인들, 다른 심리 단계
- 베드로 3번 부인(요 18장) → 3번 고백(요 21장) = 동일 인물, 심리 전환

**기존 TYPE-B와의 차이:**
> TYPE-B = 다른 기자가 다른 시점을 기록
> TYPE-B-ψ = 동일 기자·동일 인물의 심리 상태 변화로 행동이 달라진 것

---

### TYPE-B-μ: 미시 자세 변화 (Micro-Sequential State Change)

```
발동 조건: 동일 장면 내에서 동일 주체의 자세/위치가 짧은 시간 내 변화한 경우
원리:      복음서 기자마다 장면의 "어느 순간"을 포착했는지 달라
           초/분 단위의 미시적 시간차가 존재
```

**적용 사례:**
- 눅 24:4 "STOOD BY" (서서 등장) → 막 16:5 "SITTING" (앉아서 말씀)
- 헬라어 검증: ἐπέστησαν (순간 등장) → καθήμενον (현재분사, 조우 시 상태)

**기존 TYPE-B와의 차이:**
> TYPE-B = 시간 차이가 분/시간/일 단위
> TYPE-B-μ = 동일 장면 내 초 단위 상태 변화 (미시적)

---

### TYPE-B-π: 지각 필터 (Perception Filter / Witness Capacity Limit)

```
발동 조건: 증인이 사건 현장에 있었으나 심리·신체·환경적 이유로
           정보를 정상 처리하지 못한 경우
원리:      목격 ≠ 인식. 충격/애도/문화적 선입견이 지각을 차단
```

**적용 사례:**
- 막달라: 천사 메시지를 들었으나(T1-B) 애도 상태로 처리 불가 → "도난 의심" 보고(T1-C)
- 요 20:14: 막달라가 부활하신 예수님을 동산지기로 오인 (충격+첫 대면)
- 눅 24:16: 엠마오 제자들 = "눈이 가리워져" 예수님을 못 알아봄

**기존 TYPE에 없는 이유:**
> 지금까지 증인은 "본 것 vs 못 본 것"으로만 처리
> 실제로는 "봤으나 처리 불가"라는 제3 상태가 존재

## 3. IMPORT-X 프로토콜 제안 (외부 데이터 입력)

### 무덤사건에서 발생한 문제

```
사용자의 엑셀 → 이미지로 분할 전송 → AI가 수작업으로 읽음
→ 정보 손실 가능성 + 처리 시간 증가 + 자동화 불가
```

### IMPORT-X: 외부 자료 이식 프로토콜

```
IMPORT-X 발동 조건:
  - 사용자가 이미지(표/차트/텍스트) 제공
  - 사용자가 Excel/CSV 병렬 데이터 제공
  - 외부 논문/보고서 발췌 제공

IMPORT-X 실행 순서:
  Step 1: 이미지/파일 수신
  Step 2: 구조 파악 (몇 개 열? 행? 어떤 데이터?)
  Step 3: BVCAP 호환 형식으로 변환
           [열1: 시간/단계] [열2~N: 복음서/증인별 내용]
  Step 4: CHRONICLE 타임라인 초안 자동 생성
  Step 5: 사용자 검토 후 확정
```

**현재 가능 수준:** 이미지 읽기 + 수동 변환
**목표 수준:** 이미지/엑셀 → 자동 타임라인 초안 생성

## 5. 무덤사건이 보여준 새로운 사건 유형 분류

### CASE-MULTI: 다중 인물 동시 현장 분기 사건

```
기존 시스템: 단일 충돌 (A vs B)
무덤사건:    동시에 5개 인물그룹 × 4복음서 = 20개 데이터 포인트 교차

처리 방법:
  1. 인물 분리 (PERSON-SPLIT): 각 인물그룹을 독립 스레드로 처리
  2. 시간 잠금 (TIME-LOCK): 각 스레드의 시간 좌표 확정
  3. 교차 검증 (CROSS-CHECK): 스레드 간 충돌 재검증
  4. 순서 잠금 (ORDER-LOCK): 4복음서 내부 순서 역전 여부 확인
```

**이 유형은 기존 TYPE-B 단독으로 처리 불가.**
**TYPE-B + TYPE-B-ψ + TYPE-B-π + TYPE-C + DE-OVERLAP 복합 발동 필요.**
