# 🎯 GATE 분류 지침 & 레퍼런스 케이스 인덱스

> **이 파일의 목적**: 새 난제 입력 시 어떤 TYPE으로 분류할지 결정하는 GATE 지침과,
> `CHRONICLE(전례)/` 폴더의 완성 보고서 전체 인덱스를 제공한다.
>
> **공통 금지 행동**은 `CALLING(소명)/OVERRIDE_AcademicBias.md`에 통합 관리된다.

## ⚙️ GATE 분류 지침: 새 난제 입력 시 자동 분류

```
[GATE 0] 난제 유형 분류 (자동)
  → 숫자/나이/기간 충돌?          → TYPE-A 또는 TYPE-C 우선 의심
  → 두 기록이 정반대로 묘사?      → TYPE-B 우선 의심
  → 족보/나열 순서/인물 호칭 문제? → TYPE-D 우선 의심
  → 경쟁 해석이 존재?             → TYPE-E 반드시 가동
  → 구절 삭제/사본 공격?          → TYPE-G + TYPE-H 콤보 가동
  → 역사성 공격?                  → TYPE-J 우선 가동
  → "과학적으로 불가능" 공격?     → TYPE-K 가동
  → 수치/단어 반복 패턴 발견?     → TYPE-I 발동
  → 특정 인물에만 나타나는 패턴?  → TYPE-N → TYPE-F 순서로 발동
  → 복합 충돌?                    → 해당 TYPE 복수 조합 실행

[GATE 1] 관련 구절 전체 수집 (필수, 절대 생략 금지)
  → 충돌 구절 + 병행 구절 + 제3의 관련 구절 + KJV 원문 전체 나열

[GATE 2] 기존 학계 주석 검색 금지
  → 주석서/학자 견해는 GATE 4 이후 교차 검증용으로만 사용
  → GATE 2에서는 오직 성경 텍스트 데이터만으로 분석한다

[GATE 3] 해당 TYPE의 SKILL 실행
  → `QUIVER(무기고)/` 폴더의 해당 TYPE 파일을 열어 절차를 그대로 따른다

[GATE 4] 제3의 수치/사례로 역산 교차 검증
  → 결론이 나왔으면 성경의 다른 숫자나 구절로 역산하여 수학적으로 확인

[GATE 5] 마스터피스 보고서 출력 (BVCAP v1.5 양식 준수)
  → 현대 비유 1개 이상 필수 포함 (ANALOGY-5)
  → 결론보다 '증명 과정'을 더 상세하게 서술
  → 표와 타임라인을 반드시 시각적으로 제시
  → `VERDICT(판결록)/` 폴더에 `VERDICT_[난제명].md` 포맷으로 저장
```

## 🧪 복합 TYPE 교전 교범 — 요한1서 5:7 콤마 케이스 (G+H+I)

> **이 케이스는 TYPE-G, H, I 세 무기가 어떻게 연결되어 하나의 체인을 완성하는지 보여준다.**
> 복합 TYPE 분석 시 이 구조를 교범으로 삼아라.

```
[충돌] 요한1서 5:7 — "후대 삽입이다" (학계 통설 H0)

[TYPE-G 발동 — 문법 체인 검증]
  → 7절 삭제 시: 8절의 남성 분사 οἱ μαρτυροῦντες가
    중성 명사(물·피·영)를 수식 → 헬라어 性(gender) 불일치 발생
  → 문법 붕괴 = 7절은 원본에 있었다는 증거

[TYPE-H 발동 — 사본 독립성 역전]
  → 7절 포함 독립 출처 4개: 라틴 불가타 + 시리아 페쉬타 +
    스페인 교부 프리스킬리안(AD 380) + 알렉산드리아 레이아웃 구조
  → 사본 수(minority) < 독립성(4개 언어·지역·시대) → 독립성 우위

[TYPE-I 발동 — 수학적 설계 검증]
  → 창 1:27 = 22단어 (bara 3회 = 삼위일체 구조)
  → 요일 5:7 = 22단어 (3위격 구조)
  → 히브리어 알파벳 22자 = 완전한 말씀의 수
  → 수치 대칭 = 의도적 설계

[TYPE-L — 연쇄 체인 완성]
  G(문법 붕괴) + H(독립성 우위) + I(수학 설계)
  → 세 증거가 독립적으로, 동시에, 같은 방향을 가리킨다
  → 우연 불가 = 설계 확정

[판결] ✅ CONSISTENT — 유력 견해 🟡
  "7절은 문법적 필요, 역사적 증거, 수학적 설계로 삼중 입증된 원본이다."
```
