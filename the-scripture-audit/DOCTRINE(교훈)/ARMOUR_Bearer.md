## 📐 출력 품질 기준 (Output Quality Standard)

> [!IMPORTANT]
> `QUIVER(무기고)/` 의 TYPE 파일들은 **"어떻게 분석하는가"** (절차서)다.
> `CHRONICLE(전례)/` 의 완성 문서들은 **"결과물이 어떻게 생겨야 하는가"** (품질 기준)다.
> 에이전트는 보고서 작성 전, 아래 기준 문서 중 주제와 가장 유사한 것을 먼저 열어 깊이와 형식을 확인해야 한다.

### 📊 분석 케이스 분류 (Case Classification) — TYPE-B 발동 전 먼저 결정

```
CASE A: 단순 모순 (단일 복음서, 1~3구절)
  → PRE-STEP 0 (A~G) → 분석 → 자체 체크 → 출력
  → PHASE 0 (전문 인용) 불필요

CASE B: 복잡 병렬 (2개 이상 복음서, 5구절 이상, 복수 인물)
  → PHASE 0 (대상 구절 KJV 전문 인용 먼저) → PRE-STEP 0 → 분석 → 자체 체크 → 출력
  → 오늘 부활 아침 = CASE B 해당 (마/막/눅/요 4복음서)

CASE C: 신학 해석 논쟁 (의미·의도 불확실)
  → 분석 후 확정 불가 시 → ⚠️ UNRESOLVED 선언 (루프 금지)
  → 예: "behold"가 비가시성을 의미하는가 vs 서사 강조인가
```

### ✅ 출력 품질 체크리스트 (VERDICT 저장 전 반드시 확인)

```

  → 재분석 후도 불확실 → UNRESOLVED 선언 (루프 종료)

□ VERDICT(판결록)/VERDICT_[난제명].md 포맷으로 저장했는가?
□ [인물] 동명 인물 → 이름(식별자) 형식 일관 표기했는가?            ← 필수
```

## 🏷️ 인물 명칭 일관성 규칙 (Character Disambiguation Protocol)

> **동일한 이름을 가진 인물이 2명 이상 등장하면, 보고서 전체에서 반드시 식별자를 병기한다.**
> "마리아", "야고보" 같은 단독 이름은 금지. 처음 등장부터 끝까지 `이름(식별자)` 형식 유지.

### 표기 형식

```
이름(식별자)     ← 동명 인물
이름(이명)       ← 같은 사람, 다른 이름
```

> 📖 **전체 식별자 목록, 이명 사전, 음역 대조표, 족보 법칙, 지명 대조표:**
> → `DOCTRINE(교훈)/LEXICON_Bible.md` 참조

> [!CAUTION]
> **"다른 마리아", "또 다른 야고보", "어떤 유다" 같은 표현은 분석 문서에서 사용 금지.**
> 식별자 미확정 시: `이름(미확정)` 표기 후 TYPE-M 발동 검토.
> 🔴 동시대 동명 인물 — 식별자 없으면 분석 전체 오염
