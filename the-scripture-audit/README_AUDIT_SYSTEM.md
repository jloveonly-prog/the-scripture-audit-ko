# 🏛️ THE SCRIPTURE AUDIT SYSTEM
**"Search the scriptures" — John 5:39 KJV**

> **버전**: BVCAP v2.0 + AIDD Unified v2.7
> **상태**: FULLY OPERATIONAL
> **핵심 철학**: AI는 주석서를 검색하지 않는다. 원문을 분해하고, 숫자를 역산하고, 체인을 완성한다.
> **BVCAP = BALANCE(저울)** — *"Thou art weighed in the balances"* Dan 5:27

## 📜 시스템의 기원: 고대 유대교 해석학(Midrash)의 디지털 복원

본 시스템의 28종 포렌식 무기(QUIVER)는 현대의 어설픈 변증이 아닙니다. **1세기 랍비 힐렐의 7대 원칙(Middot)**을 비롯해, 토라를 필사하던 고대 히브리인들의 순정 성경 해석학을 AI 로직으로 1:1 완벽하게 부활시킨 **'디지털 산헤드린'**입니다.

| 고대 유대교 해석학 (랍비 힐렐의 7대 규칙) | BVCAP 매칭 무기 | 작동 원리 |
|:---|:---:|:---|
| **1. 칼 바호메르 (Kal Vahomer)** <br> 가벼운 것과 무거운 것의 확장 | **TYPE-L** (연쇄 추론) <br> **TYPE-P** (역논법) | 작은 단서 하나로 거대한 결론을 연쇄 증명하거나, 가벼운 억지 주장을 무거운 신성에 대입해 스스로 파괴되게 함 |
| **2. 게제라 샤와 (Gezera Shawa)** <br> 동등한 어휘의 다리 연결 | **TYPE-S** (어휘 교차 연결)<br> **TYPE-I** (어휘 빈도) | 두 구절에 쓰인 정확히 같은 '희귀 단어'를 다리(Bridge)로 삼아 모순처럼 보이는 사건을 하나로 묶어냄 |
| **3. 빈얀 아브 미카투브 에하드** <br> 한 구절에서 도출된 대전제 | **TYPE-U** (첫 언급의 법칙)<br> **TYPE-M** (이상 감지) | 단어가 성경에 **'처음'** 등장한 앵커(Anchor) 구절의 용법을 대전제로 삼아 이후 모든 해석의 변질을 차단 |
| **4. 빈얀 아브 미셰네 케투빔** <br> 두 구절에서 도출된 대전제 | **TYPE-B** (순차 병렬 통합)<br> **TYPE-Y** (다중 목격자) | 충돌해 보이는 두 구절 중 하나를 버리는 것이 아니라, 초 단위 타임라인으로 결합하여 더 거대한 뼈대를 건축 |
| **5. 켈랄 우프라트 (Kelal U-Prat)** <br> 포괄적 묘사와 구체적 디테일 | **TYPE-C** (기능 범주 분리)<br> **TYPE-W** (예언적 원근법) | 큰 뭉텅이의 묘사(예: 격납고 4만)와 세부 단위(전차 4천)를 기능과 범주로 엄격히 분리하여 모순 해방 |
| **6. 카요제 보 (Kayoze Bo)** <br> 다른 구절을 통한 유추/평행 | **TYPE-F** (예표 삼중 구조)<br> **TYPE-X** (카이아즘 구조) | 다른 곳에 있는 유사한 구조(그림자와 실체)나 히브리 특유의 대칭 구조를 엑스레이로 촬영해 문맥을 복원 |
| **7. 다바르 할라메드** <br> 문맥과 흔적을 통한 설명 | **TYPE-G** (문법 구조 해부)<br> **TYPE-O** (물리적 흔적 부재) | 단어의 문법적 배열(콤마, 관사)과 물리적 흔적의 유무를 추적하여 변개 공격과 과거 성취론을 방어 |

> 💡 **의의:** *이슬람/자유주의 신학이 성경을 "헬라적/서구적 잣대"로 읽으며 모순이라 공격할 때, 본 시스템은 "성경을 기록한 히브리인들의 원래 방식"으로 코드를 해독하여 공격의 근원 자체를 무력화시킵니다.*

## ⚖️ BVCAP 엔진의 듀얼 시스템 (1.0 vs 2.0)

본 시스템은 공격의 난이도와 목적에 따라 두 가지 버전의 검증 엔진을 선택하여 가동합니다.

| 엔진 버전 | 파일명 | 목적 및 특징 | 연동 범위 |
|:---:|:---|:---|:---|
| **v1.0** | `BVCAP_1.0.md` | **독립적/경량화 검증 (Standalone)** | 외부 무기고(QUIVER)에 의존하지 않고, 단일 구절이나 비교적 단순한 난제를 자체 내장된 6단계 로직으로 신속하게 판결할 때 사용합니다. |
| **v2.0** | `BVCAP_2.0.md` | **하이브리드 모의 법정 (Heavy-duty)** | 이슬람/무신론자들의 매우 복잡하고 난해한 공격을 방어할 때 사용합니다. 비판자와 분석자가 10라운드 치열한 토론을 벌이며, **반드시 28종의 무기고(QUIVER)를 총동원**하여 타격하고 Pipeline을 거쳐 출력합니다. |

## ⚙️ 코어 엔진 프로토콜 (Core Protocols)

무기(TYPE)를 꺼내기 전, BVCAP 엔진이 강제하는 4대 기본 법칙입니다.

| 프로토콜 | 이름 | 기능 및 목적 |
|:---:|:---|:---|
| **OVERRIDE-0** | AI 편향 거부 | 학계 통설을 Hypothesis-0으로 격리하고, 오직 KJV 원문만으로 0점 영점 조준 |
| **ANCHOR-1** | 제3 앵커 수집 | 충돌하는 두 구절 외에, 제3의 독립 데이터를 수집하지 않으면 연산을 시작하지 않음 |
| **DE-OVERLAP** | 중첩 해체 규칙 | 사건의 시간·공간을 절대 겹치지 않고 무조건 '직렬(Sequential)' 분리로 배치 |
| **MATRIX-3** | 매트릭스 역산 | 직렬 배치된 데이터를 표에 대입하여, 수치와 맥락이 100% 정합할 때까지 연산 반복 |

## 📊 무기고(QUIVER) 빠른 참조 맵 (28종 TYPE 무기)

실제 `QUIVER` 폴더에 `TYPE-*.md` 파일로 탑재되어 있는 28종의 정밀 포렌식 무기입니다.

| 무기 (TYPE) | 이름 | 한 줄 요약 |
|:---:|:---|:---|
| **TYPE-A** | 연대기 직렬 분해 | 숫자를 순서대로 줄 세워 숨겨진 연수를 역산 |
| **TYPE-B** | 사건 순차 병렬 통합 | 두 기록을 타임라인으로 끼워 맞춰 단일 서사로 통합 |
| **TYPE-C** | 기능적 범주 분리 | 같은 단어가 다른 규모/기능을 가리킴을 분해 |
| **TYPE-D** | 히브리 서사 관습 역이용 | 중요도 우선 나열·족보 공식으로 모순 해소 |
| **TYPE-E** | 경쟁 모델 전수 기각 | 모든 대안을 나열하고 하나씩 기각, 생존 모델 채택 |
| **TYPE-F** | 예표 삼중 평행 구조 증명 | 예표→실체→계승의 3단 구조를 표로 대조 증명 |
| **TYPE-G** | KJV 접속사/문법 구조 해부 | 콤마·접속사·관사로 숨겨진 구조와 삭제 불가성 증명 |
| **TYPE-H** | 사본학적 증거 독립성 역전 | 사본 수 열세를 독립 출처 수로 역전 |
| **TYPE-I** | 어휘 빈도 대칭 설계 검증 | 동사/명사 사용 횟수의 정확한 대칭으로 설계 의도 증명 |
| **TYPE-J** | 외부 역사 문헌 교차 검증 | 비성경 역사서로 성경 주장의 역사성 입체 강화 |

## 🚀 사용 방법 (How to Use)

### 📌 방법 1: 빠른 단발 공격 방어

```
"[난제 내용]을 검증해줘.
1. CALLING(소명)/Persona_Scribe42.md — 페르소나 장착
2. DOCTRINE(교훈)/ARMOUR_Bearer.md — 전투 순서 확인
3. CHRONICLE(전례)/에서 동일 TYPE 전례 확인
4. QUIVER(무기고)/TYPE-X.md — 해당 무기로 분석 실행
5. BVCAP_1.0.md — BALANCE 검증 후 판결 선고
완료 후 VERDICT(판결록)/에 마스터피스 양식으로 저장해."
```

**예시:**
> "INBOX의 Shabir Ally 공격 중 Q01(다윗 인구조사)을 검증해줘.
> QUIVER/ARMOUR_Bearer.md 파이프라인을 따르고,
> VERDICT/VERDICT_Q01_다윗인구조사.md로 저장해."

---

### 📌 방법 2: 특정 무기(TYPE) 지정 방어

`INBOX/Bible_Defense_List.md`에서 TYPE이 매핑된 항목은 직접 지정 가능.

```
"[난제]를 TYPE-[X] 무기로 검증해줘.
QUIVER/TYPE-[X]_[이름].md 절차를 따르고,
CHRONICLE/[X]_[유사난제].md를 품질 기준으로 참고해서
VERDICT/에 마스터피스 보고서로 저장해."
```

**예시 (숫자 충돌):**
> "아하지야 22세 vs 42세 난제를 TYPE-A 무기로 검증해줘.
> `QUIVER/TYPE-A_Chronological.md` 절차를 따르고,
> `CHRONICLE/[A]_아하지야_42vs22.md`를 품질 기준으로 참고해."

**예시 (사본 공격):**
> "요한1서 5:7 콤마 구절 삭제 공격을 TYPE-G+H+I 복합 무기로 검증해줘.
> `CHRONICLE/[G+H+I]_요한1서_콤마.md`를 품질 기준으로 참고해."
