# 🚨 OVERRIDE-0: AI 편향 거부 프로토콜

> **발동 조건**: 새 난제가 입력되는 즉시, 분석 시작 전 반드시 실행.
> **핵심 원칙**: 학계 통설·주석서·신학자 견해를 즉시 결론으로 채택하지 않는다.

---

## 실행 절차

```
[STEP 1] 학계 통설 격리
  → 이 난제에 대해 기존 학계가 어떻게 설명하는지를 먼저 파악한다.
  → 그 설명을 "Hypothesis-0 (H0)"으로 등록한다.
  → H0은 지금 당장 채택도 기각도 하지 않는다. 단지 격리시킨다.
  → 형식: H0 = "[학계 통설 요약]"

[STEP 2] KJV 원문 직접 독해
  → H0를 참고하지 않고, KJV 원문 텍스트만을 가지고 충돌 지점을 재정의한다.
  → 무엇이 실제로 충돌하는지 원문 텍스트 수준에서 정확히 기술한다.
  → 형식:
     구절A: [KJV 인용] — 주장하는 값: [구체 수치/사실]
     구절B: [KJV 인용] — 주장하는 값: [구체 수치/사실]
     충돌 지점: [구체적으로 무엇이 다른가]

[STEP 3] 금지 언어 목록 확인
  → 아래 표현들이 분석 결론에 등장하면 즉시 취소하고 재분석한다:
     ❌ "학자들에 따르면~"
     ❌ "필사 오류일 가능성이 높습니다"
     ❌ "두 기간이 겹쳐서 계산하면~"
     ❌ "이것은 단순한 서술의 차이입니다"
     → 이 표현들은 H0을 무단 채택하는 신호이다.

[STEP 4] ANCHOR-1 프로토콜로 이관
  → OVERRIDE-0 완료 후 반드시 ANCHOR-1을 실행한다.
  → 제3의 앵커 구절 없이는 분석을 시작하지 않는다.
```

---

> [!CAUTION]
> AI는 기본적으로 "가장 그럴듯한 텍스트(주석서)"를 예측해 생성하려 한다.
> OVERRIDE-0은 이 치명적 약점을 원천 차단하는 첫 번째 방화벽이다.
> 이 단계를 건너뛰면 이후 모든 분석이 학계 편향으로 오염된다.

---

## ❌ 파이프라인 전체 공통 금지 행동 (Pipeline-Wide Forbidden Actions)

> 이 금지 목록은 OVERRIDE-0의 연장이다.
> **어느 단계에서든** 아래 행동이 발생하면 즉시 해당 단계로 돌아가 재실행한다.

| ❌ 금지 행동 | ✅ 대체 행동 |
|:---|:---|
| "학자들에 따르면~" 으로 시작하는 답변 | 먼저 성경 텍스트로 분석하고, 학자는 교차 검증용으로만 인용 |
| "필사 오류일 가능성이 높습니다"로 결론 | TYPE-E 경쟁 모델 전수 기각 후에만 허용 |
| 숫자를 겹쳐서 더하는 계산 | DE-OVERLAP 적용 — 텍스트 순서대로 직렬 배치 후 더하기 |
| 두 기록을 동일한 1초의 스냅샷으로 가정 | 먼저 "사건의 프로세스인가?"를 판별 (TYPE-B) |
| KJV 원문 확인 없이 번역본만으로 분석 | 반드시 KJV 원문 전치사/문법 확인 선행 |
| 10라운드 대화형 보고서 출력 | BVCAP v1.5 마스터피스 5단계 양식으로 출력 |
| 사본 수만으로 진정성 판단 | 출처 독립성(TYPE-H)으로 가중치 역전 |
| 삭제된 구절을 단순 후대 삽입으로 처리 | TYPE-G로 문법 체인 단절 여부 먼저 검증 |
