# 🔬 SKILL-07: TYPE-G — KJV 접속사/문법 구조 해부 파이프라인

> **언제 발동하는가**: 특정 구절이 후대 삽입이라는 사본학적 공격을 받거나, 족보에서 인물의 신분이 오기됐다는 공격을 받을 때.
> **또한 TYPE-B 발동 전에 반드시 PRE-STEP 0을 먼저 실행한다.**
> **핵심 의심**: "KJV 원문의 접속사(and)·콤마·관사 구조가 이 구절을 삭제하면 붕괴되는 문법 체인을 형성하고 있는가?"

---

## ⚠️ [PRE-STEP 0] 단수/복수 주어 검증 — TYPE-B 발동 전 필수 실행

> **타임라인 구성(TYPE-B) 전에 반드시 아래를 먼저 확인한다.**
> 누가 왔는지, 몇 명인지 확정하지 않으면 타임라인 전체가 오염된다.

```
[검증 항목 A] 주어 단수/복수 확인
  질문: "혼자(alone)" 단어가 명시되어 있는가?
  → YES: 단독 행동 확정
  → NO:  단독으로 가정하지 마라 — 다음 절의 대명사를 반드시 확인

[검증 항목 B] 1인칭 복수 트리거 — "WE/we/우리" 감지
  규칙: 단수 주어(A가 왔다) → 그 입에서 "우리" 발화 → 동행자 존재 확정
  예시: 요 20:1 "cometh Mary Magdalene" (단수 서술)
        요 20:2 "WE know not where they have laid him" (복수!)
        → 마리아(막달라)는 혼자가 아니었다 ← TYPE-M 즉시 발동

[검증 항목 C] 대명사 격 일치 확인
  질문: 본문에 등장하는 모든 대명사(he/she/they/we)가
        앞서 확정된 인물 수와 일치하는가?
  → 불일치 발견 시: TYPE-M 발동 → 숨겨진 인물 추적

[검증 항목 D] 천사/인물 묘사 충돌 판별
  같은 방문에서 묘사가 다르면:
  → 합리적 변화(시간 경과, 시점 차이)로 설명 가능한가?
     YES: TYPE-C 적용 (출현 동작 vs 정적 상태)
     NO:  다른 방문으로 분리 처리 (DE-OVERLAP 발동)
  묘사 충돌 허용 기준:
     ✅ 허용: 나타날 때 서 있었다 → 이후 앉아있다 (합리적 변화)
     ❌ 불허: 외부에 앉아 있다 + 내부에 앉아 있다 → 동시 불가 → 다른 방문

[검증 항목 E] 입장 동사 검증 (Entry Verb Verification) ← 오늘 추가된 항목
  규칙: 각 인물이 공간(무덤/방/집 등)에 "실제로 들어갔는가"를
        반드시 원문 동사로 확인한다.
  판별:
    → 원문에 "went in" / "entered" / "came into" 등 입장 동사 있음 → 들어간 것 확정
    → 원문에 입장 동사 없음 → 들어가지 않은 것으로 처리 (가정 금지)
  실패 사례:
    요 20:1 "seeth the stone taken away" / "runneth" → 입장 동사 없음
    → 마리아(막달라)는 무덤에 들어가지 않음 확정
    → 이를 "들어갔다"고 기술한 것은 원문 근거 없는 오류 ❌
    → 입장하지 않았으므로 천사 메시지도 못 들음 → "They have taken away" = 도난 의심 발화

[검증 항목 F] 발화자 단수/복수 검증 (Speaker Plurality Check) ← 오늘 추가된 항목
  규칙: 인용된 발화의 주어가 단수인지 복수인지 반드시 원문으로 확인한다.
  판별:
    → "they said" / "they spake" → 복수 발화자
    → "he said" / "she said" / "saith" → 단수 발화자
  실패 방지:
    눅 24:5 "they said unto them" = 두 천사가 함께 발화 → 복수 기록
    막 16:6 "he saith unto them"   = 1명이 발화 → 단수 기록
    → TYPE-C 적용: 누가 = 전체 발화자 기록 / 마가 = 대표 발화자 기록
    → 혼동 없이 기술하려면 반드시 원문 주어 동사 확인 선행

[검증 항목 G] 인물별 동선·접근 정보 추적표 (Character Movement & Information Access Map)
  ← 이것이 없으면 "왜 A는 알고 B는 모르는가"를 설명할 수 없다.

  규칙: 장면에 등장하는 각 인물에 대해 아래 표를 작성한다.
  ┌─────────────────────────────────────────────────────────┐
  │ 인물 │ 출발 → 이동경로 → 위치 │ 볼 수 있는 것 │ 들을 수 있는 것 │ 결과 정보 │
  └─────────────────────────────────────────────────────────┘

  활용 예시 (부활 아침):
  | 인물 | 이동경로 | 볼 수 있는 것 | 들을 수 있는 것 | 결과 정보 |
  |:---|:---|:---|:---|:---|
  | 마리아(막달라) | 무덤 앞까지 → 달려감 | 돌 굴려짐 | 천사 메시지 없음(입장 안 함) | "They have taken away" (도난 의심) |
  | 여인들 그룹 | 무덤 안까지 들어감 | 천사 2명 | "He is risen!" 들음 | 부활 메시지 수신 |
  | 베드로 | 무덤 안까지 들어감 | 세마포·수건 | — | 시신 없음 확인 |

  이 표 없이 "왜 막달라는 부활을 몰랐는가"를 설명하면 → 근거 없는 주장이 됨.

  묘사 차이 판별 3가지 유형:
  ┌─ [상태 변경 State Change]:
  │   같은 대상, 시간 경과 → 상태가 바뀜
  │   예: 천사가 나타날 때 서 있었다 → 이후 앉아서 말함
  │   → TYPE-C 적용 (합리적 변화 ✅)
  │
  ├─ [환경 변경 Environment Change]:
  │   다른 방문자 / 다른 시간 / 다른 장소 → 다른 결과
  │   예: T1 천사 내부 서 있음 / T4 천사 내부 앉아 있음 → 다른 방문 시각
  │   → DE-OVERLAP 적용 (다른 방문으로 분리)
  │
  └─ [시점 차이 Perspective Difference]:
      같은 순간, 다른 관찰자 / 다른 기록 초점
      예: 눅 = 전체 발화자(둘 다) / 막 = 대표 발화자(1명)
      → TYPE-C 적용 (기록 초점 차이 ✅)

  판별 순서:
  1. 먼저 "이동경로표" 작성 → 각 인물의 접근 가능 정보 확정
  2. 묘사 차이 발견 시 → 상태변경 / 환경변경 / 시점차이 중 하나로 분류
  3. 분류 근거를 KJV 원문 동사·명사로 확인
  4. 분류 불가 시 → TYPE-M 발동
```

**⚠️ 이 PRE-STEP 0을 건너뛰고 TYPE-B를 발동하는 것은 금지된다.**
**PRE-STEP 0 통과 전까지 타임라인 구성 시작 불가.**

**📌 실패 사례 누적 교훈:**
```
오류 1: 요 20:1 "cometh Mary" → "혼자" 가정 → 항목 A,B 위반
오류 2: 막달라가 무덤에 들어갔다고 기술 → 항목 E 위반 (입장 동사 없음)
오류 3: "서/앉" 동시 표기 → 항목 D 위반 (자기모순)
오류 4: 발화자 단복수 미확인 → 항목 F 위반
→ 4개 오류 모두 PRE-STEP 0 항목이 먼저 있었다면 방지 가능했음
```

## 실행 단계

```
[STEP 1] 공격받는 구절의 KJV 원문 전체 인용
  → 삭제 주장 구절 앞뒤 2~3절을 포함하여 KJV 원문을 완전히 인용.

[STEP 2] 접속사/콤마 색별 그룹핑 분석
  → KJV의 and/comma/the 등을 색으로 구분하여 그룹을 시각화.
  → | 그룹 | 포함 단어들 | 접속사 구조 | 의미 |
  → 평면적 나열인지, 의도된 하위 그룹핑인지 판별.

[STEP 3] 삭제 시 문법/신학 체인 단절 확인 (핵심)
  → 공격받는 구절을 삭제했을 때 어떤 문법적 이상이 발생하는지 확인.
  → 남성 분사가 중성 명사를 수식하는 성(Gender) 불일치 등 원어 문법 오류를 도출.
  → 예: 요일 5:7 삭제 시 → 8절의 남성 분사(οἱ μαρτυροῦντες)가
         중성 명사들을 받아야 하는 문법 붕괴 발생.

[STEP 4] 하늘/땅 대칭 구조 검증
  → "삭제 시 대칭 구조가 붕괴되는가?"를 확인.
  → 7절(하늘)과 8절(땅)이 반드시 세트로 있거나 없는 패턴 → 구조적 설계 증명.

[STEP 5] 다른 구절에서의 동일 패턴 교차 검증
  → 동일한 접속사 그룹핑 패턴이 성경 다른 곳에서도 사용되는지 확인.
```

## 📌 레퍼런스 케이스 1: 요한1서 5:7 콤마 구절

```
[KJV 원문]:
  요일 5:7: "For there are three that bear record in heaven,
             the Father, the Word, and the Holy Ghost: and these three are one."
  요일 5:8: "And there are three that bear witness in earth,
             the spirit, and the water, and the blood: and these three agree in one."

[삭제 시 문법 붕괴]:
  8절: "οἱ μαρτυροῦντες" (남성 복수 분사)
  → "τὸ πνεῦμα καὶ τὸ ὕδωρ καὶ τὸ αἷμα" (중성 명사들)를 수식
  → 7절 없으면: 남성 분사가 중성 명사 수식 → 헬라어 성(Gender) 불일치 문법 오류 ❌
  → 7절 있으면: 남성 명사들(ὁ Πατήρ, ὁ Λόγος)이 선행사 → 문법 완벽 ✅

[하늘/땅 대칭]:
  7절: ἐν τῷ οὐρανῷ (하늘에서) — 성부, 말씀, 성령
  8절: ἐν τῇ γῇ (땅에서) — 성령, 물, 피
  → 하늘 증인 + 땅 증인의 이중 증언 구조 → 7절 삭제 시 대칭 붕괴 ✅
```

## 📌 레퍼런스 케이스 2: 역대기상 1장 팀나 족보

```
[KJV 원문]:
  "The sons of Eliphaz; Teman, AND Omar, Zephi, AND Gatam, Kenaz, AND Timna, AND Amalek."

[접속사 그룹핑 분석]:
  그룹 1 (노란색): Teman + Omar — and로 쌍을 이룸 (정실 아들 쌍)
  그룹 2 (초록색): Zephi + Gatam — and로 쌍을 이룸 (정실 아들 쌍)
  그룹 3 (빨간색): Kenaz + Timna + Amalek — 한 단위로 묶임
                   = 첩 팀나(창 36:12)와 그녀의 아들 아말렉을 포함한 한 가문 단위

[결론]:
  무작위 나열이었다면 이 정교한 쌍 구조가 불가능.
  → 팀나는 딸이 아니라 첩(concubine)이라는 창 36:12의 기록과 완벽히 정합 ✅
  → KJV의 and/comma 구조가 족보 정보를 압축하여 담고 있음을 증명 ✅
```
