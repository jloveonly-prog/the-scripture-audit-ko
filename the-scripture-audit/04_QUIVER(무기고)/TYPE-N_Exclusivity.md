# 🔬 SKILL-14: TYPE-N — 배타성 검증 파이프라인

> **언제 발동하는가**: TYPE-M이 "한 사람/사건에만 적용되는 이상 신호"를 감지한 이후.
> **핵심 질문**: "이 패턴이 정말로 이 사람/사건에만 적용되고 다른 경우에는 없는가?"

---

## 실행 단계

```
[STEP 1] 전수 조사 범위 설정
  → 동일한 패턴/표현이 등장할 수 있는 모든 관련 구절을 수집
  → 예: "요나의 아들" → 복음서 전체에서 이 표현이 쓰인 모든 구절 조사

[STEP 2] 전수 조사 실행
  → 수집된 모든 구절에서 해당 패턴이 등장하는지 확인
  → | 대상 인물/사건 | 해당 패턴 존재 여부 | 구절 |

[STEP 3] 배타성 판정
  → 오직 한 대상에게만 있고 다른 모든 대상에게는 없으면 → 배타성 확정 ✅
  → 둘 이상에게 있으면 → 배타성 기각, 다른 설명 필요 ❌

[STEP 3-B] 지우개 법칙 (Eraser Law) — 배타성 역검증 서브루틴
  → 대안 해석(X)을 성경 본문에 대입했을 때,
     해당 구절의 특정 단어·약속·구별의 존재 이유가 소거(erase)된다면
     → 해석 X는 거짓이다.
  → 공식: IF 해석X 대입 → 본문의 단어W가 불필요해짐 → THEN 해석X = FALSE
  → 전투기록: 요 13:36 = 천국(요 14:2)이면 → 14:3에서 모든 제자를 영접
     → 베드로만 "나중에 따라오리라"는 특별 약속의 존재 이유가 완전히 소거됨
     → ∴ 13:36 ≠ 천국 → 배타적 지상 장소(갈보리) 확정
  → 이 법칙은 STEP 3 배타성 판정과 독립적으로 발동 가능한 범용 거짓 해석 즉살 공식이다.

[STEP 3-C] 제거 테스트 (Removal Test) — 지우개 법칙 역방향
  → 지우개 법칙(STEP 3-B)의 역방향 검증이다.
  → 논증에 사용된 핵심 단어(W) 또는 구절(Y)을 본문에서 제거했을 때,
     원문의 문법·논리·배타성 체계가 동작 불가하면 → W/Y는 필수 구성 요소이다.
  → 공식: IF W제거 → 문법/논리/배타성 붕괴 → THEN W = ESSENTIAL (필수)
  → 성경 근거: "여기서 조금, 저기서 조금" (사 28:10 / Isaiah 28:10 KJV)
     → 작은 단어 하나도 제거하면 전체 설계가 무너진다 = 말빔/아키바 철학의 수학적 검증

  → 지우개 법칙(순방향)과 제거 테스트(역방향)의 관계:
     순방향 (STEP 3-B): 잘못된 해석을 넣으면 단어가 죽는다 → 해석이 거짓
     역방향 (STEP 3-C): 필수 단어를 빼면 시스템이 죽는다 → 단어가 필수
     → 동전의 양면: 둘 다 통과하면 해당 단어/구절의 원본 필수성이 이중 확정된다.

  → 실행 방법:
     ① 논증에 사용된 핵심 단어/구절 목록화
     ② 각 단어/구절을 하나씩 제거
     ③ 제거 후 원문이 동작하는지 3가지 층위에서 판정:
        - 문법 층위: 성(Gender)/수(Number)/격(Case) 일치가 유지되는가?
        - 논리 층위: 앞뒤 문맥의 논리적 흐름이 유지되는가?
        - 배타성 층위: 특정 대상에만 적용되는 구별이 유지되는가?
     ④ 하나라도 붕괴 → W = ESSENTIAL (필수) ✅
        모두 유지 → W = OPTIONAL (선택적) — 논증에서 제외 권고

  → 전투기록 1: 요일 5:7 전체 제거 → 8절 남성 분사(οἱ)가 중성 명사를 수식 → 성(Gender) 붕괴
     → 요일 5:7 = ESSENTIAL ✅ (TYPE-G와 교차 확인)
  → 전투기록 2: 요 13:36 ἄρτι(지금은) 제거 → "따라올 수 없다"에서 시간 제약 소멸
     → 베드로가 즉시 따라갈 수 있게 됨 → "나중에"의 의미 소멸 → 논리 붕괴
     → ἄρτι = ESSENTIAL ✅
  → 전투기록 3: 요 13:36 σύ(너는, 강조) 제거 → 배타성 소멸 → 모든 제자에게 해당
     → 베드로 특별 약속의 존재 이유 소멸 → 배타성 붕괴
     → σύ = ESSENTIAL ✅

[STEP 4] 배타성의 의미 해석
  → 배타성이 확정되면:
     "왜 오직 이 한 사람/사건에만 적용되는가?"
  → 이 질문이 TYPE-F(예표 구조) 또는 TYPE-L(연쇄 추론)의 입력이 된다

[STEP 5] 배타성 + 예표 연결 판정
  → 배타성 + 구약 예표 인물이 존재 → TYPE-F 즉시 발동
  → 배타성 + 수치/설계 패턴 → TYPE-I 또는 TYPE-L 발동
```

## 📌 레퍼런스 케이스: 베드로 — "요나의 아들"

```
[전수 조사]: 복음서 전체에서 "요나의 아들" 표현 탐색

  | 인물   | 해당 패턴      | 구절       |
  |:------|:------------|:---------|
  | 시몬   | 있음 ✅       | 마 16:17  |
  | 시몬   | 있음 ✅       | 요 1:42   |
  | 안드레 | 없음 ❌       | —        |
  | 야고보 | 없음 ❌       | —        |
  | 요한  | 없음 ❌       | —        |
  | (12제자 전원 조사) | — | 베드로에게만 해당 |

[배타성 판정]: 확정 ✅

[의미 해석]:
  "왜 베드로에게만인가?"

  구약 요나:   갈릴리 출신 / 물고기 / 3일 / 서쪽 이방 도시로 파송됨
  예수님:      갈릴리 출신 / 3일 무덤 / 서쪽으로 복음 확산
  베드로:      갈릴리 출신 / 요나처럼 따라가야 할 예언적 여정

  → 예수님이 "요나의 아들"이라고 부른 것
    = 요나의 여정(이방인을 향한 순종과 파송)을 따를 자라는 예언적 선언

[연결 발동]:
  → TYPE-F 발동: 요나-예수-베드로 삼중 예표 구조 전체 분석 개시
  → 갈보리(예루살렘) 순교설의 성경적 근거로 연결
```
