# 🏹 SKILL-39: TYPE-AL — 어의 중의성 적발 (Equivocation Detection)
**"같은 단어를 다른 의미로 슬쩍 바꾸는 오류를 잡아낸다"**

> **언제 발동하는가**: 상대가 논증 중 동일한 단어를 서로 다른 의미로 사용하여 혼란을 일으킬 때.
> 특히 "명시", "증거", "전통" 같은 핵심 용어의 정의를 맥락에 따라 바꿀 때.
>
> **한 줄 정의**: *"같은 단어, 다른 의미 — 그 순간 논증이 무너진다."*

---

## 발동 조건
`
[범용]
1. 상대가 논증 중 핵심 용어의 정의를 바꿀 때
   (예: "명시", "증거", "전통", "영감", "기적", "구원" 등)
2. 같은 단어를 유리할 때는 A의미, 불리할 때는 B의미로 사용할 때
3. 용어 정의 이동으로 논증의 기반이 흔들릴 때
4. [자기검증 — v1.1 추가] **아군(BVCAP) 자신의 논증**이 KJV의 동일 어근을 대문자(고유 칭호)/소문자(일반 범주)로 구분한 것을 무시하고, 한쪽 용례의 결론을 다른 쪽에 그대로 적용할 때 → 상세 절차는 아래 TYPE-AL-cap 서브 프로토콜

[전형적 코드] 아래 패턴 감지 시 즉시 발동:
  - 동일 단어가 논증 전반부와 후반부에서 다른 의미로 쓰임
  - "그것은 달라요"라고 하지만 논리 구조는 동일
  - 상대가 자신에게 유리한 사례에는 넓은 정의, 불리한 사례에는 좁은 정의 적용
  - [자기검증] "히 1:5의 Son(즉위 선언)이므로 욥 38:7의 sons(창조 범주)도 자동 배제된다"류의 대문자→소문자 자동 전이
`

## 실행 단계
`
[STEP 1] 문제 단어 식별
  → 논증에서 반복 사용되는 핵심 용어를 추출한다.

[STEP 2] 사용 맥락별 의미 분석
  → 같은 단어가 각 발화 맥락에서 어떤 의미로 쓰였는가?

[STEP 3] 의미 전환 지점 적발
  → 의미가 슬쩍 바뀐 정확한 지점을 지목한다.

[STEP 4] 기각 선언
  → "이 논증은 동일 단어를 다른 의미로 사용한 어의 중의성 오류다. 기각한다."
`

## 📌 레퍼런스 케이스: "명시" 정의 이동

| 발화 시점 | "명시"의 정의 | 목적 |
|:---|:---|:---:|
| 반론 초기 | "단일 구절에 직접 선언된 것" | 갈보리설 기각 |
| 삼위일체 반례 제시 후 | "여러 구절의 구조적 연결도 명시" | 삼위일체 구하기 |
| 다시 갈보리설 공격 시 | "단일 구절 선언이 없으면 명시 아님" | 다시 갈보리설 기각 |

> **TYPE-AL 판결**: "명시"의 정의가 논증 중 최소 2회 변경됨.
> 이는 어의 중의성 오류다. 일관된 정의를 요구한다.
> 일관된 정의 적용 시 — 갈보리설은 삼위일체와 동일한 구조로 명시 범주에 속한다. ✅

## 🆕 서브 프로토콜: TYPE-AL-cap — 대소문자 동어이의어 자기검증 (Capitalization Self-Check) — v1.1

> **발동 조건**: 대문자 신적 칭호(Son, Spirit, Word 등)가 등장하는 구절의 결론을, 같은 어근의 소문자 일반 용례에 그대로 적용하려 할 때. 상대 논증뿐 아니라 **아군(BVCAP) 자신의 논증에도 의무적으로 적용**한다.
> **참조**: `02_TACTICS(전술)/LEXICON_Bible.md` 섹션 7 (대문자/소문자 신적 칭호 경보표)에 등재된 동어이의어 목록.

```
[STEP 1] 대소문자 짝 식별
  → 논증에 사용된 단어가 LEXICON 섹션 7의 표에 등재되어 있는지 확인한다.
  → 없다면 새로 발견된 짝을 섹션 7에 추가 등재한다 (살아있는 문서 원칙).

[STEP 2] 원어 확인 — 진짜 구분인가, 편집적 대문자화인가?
  → 두 용례가 원어(헬라어/히브리어)에서 실제로 다른 어휘·문법 구조를 갖는가?
  → 대문자 용례가 인용하는 구약 원본문(예: 시 2:7)이 별도의 특정 발화 공식(즉위 선언 등)인가?
  → YES(실제 구분 있음) → STEP 3으로
  → NO(단어 자체는 동일, KJV 편집자의 문맥 판단일 뿐) → 대문자 표기를 "보조 신호"로 격하하고 결론의 유일 근거로 사용 금지

[STEP 3] 자동 전이 여부 판정
  → 대문자 용례의 결론이 소문자 용례에 "직접 적용"되는가, 아니면 "이 논증과 무관"한가?
  → 무관하다면 그 구절을 해당 결론의 근거 목록에서 제외한다.

[STEP 4] 판결 선언
  → "이 구절(대문자 용례)은 [특정 선언/의미]만 증명한다. [소문자 일반 범주]에 대해서는 증명하지 않는다."
```

### 📌 레퍼런스 케이스: 히 1:5 "Son" → 욥 38:7 "sons" 자동 전이 오류 (2026-07-19)

```
[STEP 1] 짝 식별: Son(히 1:5, 시 2:7 인용) vs sons(욥 1:6·38:7, בְּנֵי הָאֱלֹהִים)

[STEP 2] 원어 확인:
  히 1:5의 "Son"은 시편 2:7의 특정 즉위 선언 공식("this day have I begotten thee")을 인용.
  욥 1:6·38:7의 "sons of God"은 별개의 히브리어 관용구(בְּנֵי הָאֱלֹהִים) — 창조 범주 지시어.
  → 원어상 별개의 발화 층위 확인 (즉위 선언 vs 창조 범주 명칭)

[STEP 3] 자동 전이 판정:
  히 1:5는 "천사가 시 2:7식 즉위 선언을 받은 적 없다"만 증명.
  "욥 38:7의 sons가 천사인지 아닌지"는 히 1:5와 무관한 별개 질문.
  → 무관 → 근거 목록에서 제외

[STEP 4] 판결: 히 1:5는 "천사=아들 절대 불가"를 증명하지 않는다.
  "욥 38:7의 아들들 ≠ 천사" 결론은 유지 가능하나, 창조 시점·sang/say 어휘·
  ἀρχή 통치권 같은 독립 근거로만 성립해야 한다.
```
