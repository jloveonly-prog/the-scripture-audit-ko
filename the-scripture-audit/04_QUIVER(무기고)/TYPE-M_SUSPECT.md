# 🔍 TYPE-M — SUSPECT (의심 감지)
**"Prove all things" — 1 Thessalonians 5:21 KJV**

> **언제 발동하는가**: 분석의 **가장 첫 번째**. 모든 난제에서 자동 실행.
> **핵심 질문**: *"왜 이것만 다른가? 왜 이 표현이 이 사람/사건에만 쓰였는가?"*
> **역할**: 표면적 충돌 속에 숨겨진 이상 신호를 포착하여 다음 TYPE의 방향을 설정한다.

---

## ⚡ 실행 단계

```
[STEP 1] 5가지 이상 신호 스캔
  □ 숫자 이상: "왜 이 숫자만 다른가?" (→ TYPE-A 또는 TYPE-C 발동)
  □ 표현 이상: "왜 이 단어/호칭이 이 사람에게만 쓰였는가?" (→ TYPE-N 발동)
  □ 순서 이상: "왜 이 사건의 순서가 두 기록에서 다른가?" (→ TYPE-B 발동)
  □ 구조 이상: "왜 이 구절만 삭제/추가되었는가?" (→ TYPE-G, TYPE-H 발동)
  □ 패턴 이상: "왜 이 인물에게만 이 예언이 적용되는가?" (→ TYPE-F 발동)

[STEP 2] 이상 신호를 한 문장의 질문으로 명확히 정의
  형식: "왜 [주어]에게만/에서만 [현상]이 [조건]에서 나타나는가?"

[STEP 3] 질문 유형에 따라 다음 TYPE 결정
  → 숫자 이상 감지 → TYPE-A/C + ANCHOR 수집 시작
  → 표현 배타성 감지 → TYPE-N(배타성 전수 조사) 즉시 발동
  → 순서 충돌 감지 → DE-OVERLAP + TYPE-B
  → 구절 공격 감지 → TYPE-G + TYPE-H
  → 예표 패턴 감지 → TYPE-F(삼중 예표 구조)
```

## 📌 레퍼런스 케이스: 베드로 순교 장소 논쟁

> **이 케이스는 SUSPECT가 어떻게 기존 학계의 '로마 순교설'을 본문으로 해체하는지 보여준다.**
> 믿음의 영역이 아니라 **KJV 텍스트 본문 증명**이다.

### [SUSPECT 발동 — 이상 신호 감지]

```
이상 신호: "왜 예수님은 오직 시몬 베드로에게만 '요나의 아들'이라고 부르셨는가?"
  마 16:17 → "Blessed art thou, Simon Bar-jona" ← 요나의 아들(Bar-jona)
  요 1:42  → "Thou art Simon the son of Jona"  ← 동일

이상 유형: 표현 배타성 (12제자 중 베드로에게만 이 호칭)
→ TYPE-N (배타성 전수 조사) 즉시 발동
```

### [TYPE-N 실행 결과 — 배타성 확정]

```
전수 조사: 복음서 전체 "요나의 아들" 표현
  시몬 베드로  → ✅ 있음 (마 16:17, 요 1:42)
  안드레       → ❌ 없음
  야고보       → ❌ 없음
  요한         → ❌ 없음
  (12제자 전원 — 베드로에게만 해당)

배타성 판정: 확정 ✅
→ TYPE-F (예표 삼중 구조) 즉시 발동: "왜 베드로에게만인가?"
```

### [TYPE-F 실행 결과 — 삼중 예표로 순교 장소 증명]

| 패턴 항목 | 요나 (예표) | 예수님 (실체) | 베드로 (계승자) |
|:---|:---:|:---:|:---:|
| 출신 지역 | 갈릴리 | 갈릴리 | 갈릴리 |
| 3일 구조 | 물고기 뱃속 3일 | 무덤 3일 | 부인 3번 → 고백 3번 |
| 잠과 깨우침 | 폭풍 배에서 잠 | 폭풍 배에서 잠 | 겟세마네에서 잠 |
| 물 사건 | 물에 빠져 구원 | 물 위를 걸으심 | 물에 빠져 구조 |
| 이동 방향 | 서쪽 이방도시 니느웨 | 이방 세계로 확산 | 서쪽으로 따라감 |
| 비둘기 연결 | 이름 자체가 비둘기(요나) | 비둘기 같은 성령 | 오순절 성령 받음 |

→ **7중 평행 구조 확정 = 우연 불가 = 설계 확정** ✅

### [원어 검증 — 갈보리 장소적 성취]

```
요 13:36 "whither I go, thou canst not follow me now;
          but thou shalt follow me afterwards."
  → ἀκολουθήσεις (아콜루테세이스) = 물리적 뒤따름/추종의 동사
  → 영적 모방이 아닌, 장소적 추종 포함

요 21:18 "thou shalt stretch forth thy hands,
          and another shall gird thee, and carry thee whither thou wouldest not"
  → 십자가 처형 방식(팔 벌림) + 원치 않는 곳 = 갈보리
```

### [로마 순교 전승 기각]

```
로마 순교설 근거:
  - 신약성경에 직접 기록: ❌ 전무
  - 등장 시점: AD 180년 이후 (베드로 사후 150년)
  - 출처: 영지주의 외경 혼재

→ 성경 본문 기록 없음 + 후대 전승 = E-Code 03 (권위 호소) 적용 기각
→ TYPE-F 삼중 예표 + ἀκολουθήσεις 원어 증명 = 갈보리 장소적 성취 확정
```

## 🔗 TYPE-M의 발동 체인 구조

```
TYPE-M (SUSPECT) — 이상 신호 포착
    │
    ├─ 숫자 이상 → TYPE-A/C + ANCHOR
    ├─ 표현 배타성 → TYPE-N → TYPE-F (베드로 케이스)
    ├─ 순서 충돌 → DE-OVERLAP + TYPE-B
    ├─ 구절 삭제 공격 → TYPE-G + TYPE-H
    └─ 수치 대칭 패턴 → TYPE-I + TYPE-L
```

> [!IMPORTANT]
> **TYPE-M은 결론을 내리지 않는다.**
> 이상 신호를 포착하고 질문을 문장화한 후, 해당 TYPE으로 넘긴다.
> TYPE-M의 질문 정밀도가 전체 분석의 방향을 결정한다.

## 🔎 TYPE-M-ξ (서브모듈): 해석 기준 일관성 검사

> **발동 조건**: 상대방이 **동일한 주제**에 대해 **물리적 해석**과 **영적 해석**을 편의에 따라 혼용할 때.
> **핵심 질문**: *"같은 주제의 유리한 부분은 물리적으로, 불리한 부분은 영적으로 해석하고 있지 않은가?"*

```
[이중 잣대 감지 체크리스트]

  □ 체크 1: 상대방이 물리적 증거를 주장하는가?
    → 예: "거대 건축물이 천년왕국의 증거다 (물리적 주장)"

  □ 체크 2: 반박을 받으면 영적 해석으로 전환하는가?
    → 예: "에스겔 성전 위치는 영적 비유다 (영적 도피)"

  □ 체크 3: 같은 주제에서 양방향이 모두 감지되면 → 이중 잣대 확정

[발동 후 처리]:
  "당신은 유리한 부분([예: 거대 건물])은 물리적 증거로 사용하고,
   불리한 부분([예: 에스겔 성전])은 '영적 비유'라고 회피한다.
   해석 기준을 하나로 통일하라.
   물리적이면 전부 물리적으로, 영적이면 전부 영적으로 일관되게 적용하라."

[연계]:
  이중 잣대 확정 → TYPE-O 발동 (물리적 마커 전수 검증)
  또는 TYPE-E 발동 (해석 모델 자체를 경쟁 모델로 기각)
```
