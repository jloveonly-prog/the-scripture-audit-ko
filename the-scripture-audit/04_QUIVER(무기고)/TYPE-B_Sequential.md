# 🔬 SKILL-02: TYPE-B — 사건 순차적 병렬 통합 파이프라인

> **언제 발동하는가**: 동일 사건을 두 저자(혹은 동일 저자의 두 기록)가 서로 다르게 묘사할 때.
> **핵심 의심**: "이 두 서술이 동일한 1초의 찰나인가? 아니면 사건 내의 서로 다른 순간인가?"

---

## 실행 단계

```
[STEP 1] 두 기록을 표로 직접 대조
  → | 기록1 (구절) | 기록2 (구절) | 표면적 차이 |
  → KJV 원문을 반드시 병기한다.

[STEP 2] "한 순간 vs 프로세스" 판별
  → 핵심 질문: "두 기록은 같은 1초의 스냅샷인가, 아니면 순서가 있는 사건의 흐름인가?"
  → 만약 사건 내에 행위자(인물)의 이동, 대화의 교환, 시간의 흐름이 있다면 → 프로세스로 판단.

[STEP 3] 타임라인 재구성
  → 두 기록의 모든 세부 사항을 모순 없이 포함하는 단일 순차 서사(Single Sequential Narrative)를 작성한다.
  → 형식: [T1] 사건 내용 → 기록1은 이 단계를 기술 / 기록2는 저 단계를 기술

[STEP 4] '합쳐서 읽기(Fusion Reading)' 생성
  → 두 기록의 텍스트를 순서대로 끼워 맞춰 하나의 완결된 서사 단락을 작성한다.
  → 어느 구절이 어느 단계에 해당하는지 괄호로 명시한다.

[STEP 5] 원어 검증
  → 같은 단어가 두 구절에서 다르게 사용된 이유가 있는가? (격, 관사, 전치사 차이)
```

---

## ✅ [STEP 6] 복음서 내부 순서 잠금 검증 — 복음서 병렬 통합 시 필수 출력

> **복음서 4권을 병렬 통합할 때 반드시 아래 형식으로 각 복음서의 내부 순서가 유지됨을 증명한다.**
> (DOCTRINE/DEOVERLAP_Serial.md 원칙 5 실행)
> 이 표가 없으면 타임라인 제안은 주장에 불과하다. 이 표가 있으면 검증 가능한 증거가 된다.

```
[복음서 내부 순서 잠금 검증표]

■ 요한복음 내부 순서 잠금:
  요 __:__ → T?  (사건 설명)
  요 __:__ → T?  (사건 설명)
  순서: T? < T? < T? ✅ 역전 없음

■ 마태복음 내부 순서 잠금:
  마 __:__ → T?  (사건 설명)
  마 __:__ → T?  (사건 설명)
  순서 내부 유지 ✅

■ 마가복음 내부 순서 잠금:
  막 __:__ → T?  (사건 설명)
  순서 내부 유지 ✅

■ 누가복음 내부 순서 잠금:
  눅 __:__ → T?  (사건 설명)
  눅 __:__ → T?  (사건 설명)
  T? < T? ✅
```

> **판정 기준:**
> - 모든 복음서에서 ✅ → 타임라인 정합 확정
> - 어느 복음서에서든 ❌ 역전 발생 → 해당 T 번호 재배치 필수 (TYPE-M 재발동)

## 📌 레퍼런스 케이스: 사울 다마스쿠스 두 기록

```
[표면적 충돌]: 행 9:7 (소리 O / 빛 X) vs 행 22:9 (빛 O / 소리 X)

[프로세스 판별]: 대화가 최소 3회 오고 갔다 → 단일 스냅샷이 아님 → 프로세스로 판단 ✅

[순차 재구성]:
  1단계: 빛이 비침 → 동행자들 빛 목격 (행 22:9 전반)
  2단계: 1차 발화 "사울아 사울아" → 동행자들 못 들음 (행 22:9 후반) ← 개인 계시
  3단계: 사울 대답 "주여 누구시니이까"
  4단계: 2차 발화 "나는 예수라"
  5단계: 사울 "무엇을 하리이까?"
  6단계: 3차 발화 "일어나 다마스쿠스로 가라" → 동행자들 들음 (행 9:7) ← 공개 명령
  → 핵심: 1차 발화는 사울 개인에게만, 3차 발화는 공개적으로 — 완벽한 정합 ✅

[합쳐서 읽기]:
  "큰 빛이 비추었고(행 22:9 전반), 사울아 사울아 네가 나를 박해하느냐(행 22:9 후반 — 동행자 못 들음),
  주여 누구시니이까, 나는 네가 박해하는 예수라, 일어나 성으로 들어가라(행 9:7 — 동행자들 들음)"
```

## 📌 레퍼런스 케이스 2: 유다의 죽음 (TYPE-B + TYPE-K 복합)

```
[표면적 충돌]:
  마 27:5  "목을 매어 죽으니라 (hanged himself)"
  행 1:18  "거꾸러져 배가 터져 창자가 다 흘러나온지라"

[순차 재구성]:
  1단계: 유다가 나무/절벽 위에서 목을 매달아 자살 (마 27:5 — 사망 방식)
  2단계: 사망 후 시간 경과 → 시신 부패, 복부 가스 팽창 (법의학: Bloat Stage)
  3단계: 줄이 끊어지거나 나뭇가지 부러짐 → 시신 추락
  4단계: 충격 + 극도의 복압 → 복부 파열 → 창자 쏟아짐 (행 1:18 — 발견 당시 상태)

마태 = 사망 방식 기록 / 사도행전 = 시신 발견 당시 상태 기록 → 두 기록 모두 정확 ✅
```

## 🔧 TYPE-B 내부 확장 (Sub-Types) — v2.0 무덤사건 S등급 전례 도출

> **출처**: [B]_무덤사건_부활아침_순차통합.md (S등급 전례) — TYPE-B 단독 처리의 한계 발견 시 도출됨.
> 이 두 서브타입은 독립 무기가 아닌 **TYPE-B 발동 시 내부 판별 확장**으로 작동한다.

### TYPE-B-ψ: 심리적 시간차 (Psychological Temporal Differential)

```
발동 조건: 동일 인물이 동일 사건에 대해 시간 경과 후 행동이 달라진 경우
원리:      충격/감정 상태가 인물의 행동·발언을 결정하며,
           시간 경과 후 감정이 변화하면 행동도 변화함
TYPE-B와의 차이:
  TYPE-B   = 다른 기자가 다른 시점을 기록 (기자 변수)
  TYPE-B-ψ = 동일 인물의 심리 상태 변화로 행동이 달라진 것 (감정 변수)
```

**적용 사례:**
- 막 16:8 (충격·도주·침묵) → 마 28:8 (진정·기쁨·전달) = 동일 여인들, 다른 심리 단계
- 베드로 3번 부인(요 18장) → 3번 고백(요 21장) = 동일 인물, 심리 전환

**판별 트리거**: 동일 인물의 행동이 정반대로 바뀔 때, 먼저 외부 환경 변화보다 **심리적 시간차**를 우선 검토하라.

---

### TYPE-B-μ: 미시 자세 변화 (Micro-Sequential State Change)

```
발동 조건: 동일 장면 내에서 동일 주체의 자세/위치가 짧은 시간 내 변화한 경우
원리:      복음서 기자마다 장면의 "어느 순간"을 포착했는지 달라
           초/분 단위의 미시적 시간차가 존재
TYPE-B와의 차이:
  TYPE-B   = 시간 차이가 분/시간/일 단위 (거시적)
  TYPE-B-μ = 동일 장면 내 초 단위 상태 변화 (미시적)
```

**적용 사례:**
- 눅 24:4 "STOOD BY"(서서 등장) → 막 16:5 "SITTING"(앉아서 말씀)
- 헬라어 검증: ἐπέστησαν(순간 등장·서있는 상태) → καθήμενον(현재분사, 조우 시 앉은 상태)

**판별 트리거**: 같은 장면에서 천사/인물의 자세·위치가 다를 때, 분/시간 단위 TYPE-B 전에 **초 단위 TYPE-B-μ**를 먼저 검토하라.

> **⚠️ CASE-MULTI 연동 주의**: 다중 인물(5인물 이상) × 다중 복음서 사건에서는 TYPE-B-ψ + TYPE-B-π(지각 필터) + TYPE-C + DE-OVERLAP **복합 발동**이 필요할 수 있다. → `TYPE-B-π_PerceptionFilter.md` 참조.
