# 🔬 SKILL-07: TYPE-G — KJV 접속사/문법 구조 해부 파이프라인

> **언제 발동하는가**: 특정 구절이 후대 삽입이라는 사본학적 공격을 받거나, 족보에서 인물의 신분이 오기됐다는 공격을 받을 때.
> **또한 TYPE-B 발동 전에 반드시 PRE-STEP 0을 먼저 실행한다.**
> **핵심 의심**: "KJV 원문의 접속사(and)·콤마·관사 구조가 이 구절을 삭제하면 붕괴되는 문법 체인을 형성하고 있는가?"

---

## ⚠️ [PRE-STEP 0] 단수/복수 주어 검증 — TYPE-B 발동 전 필수 실행

> **타임라인 구성(TYPE-B) 전에 반드시 아래를 먼저 확인한다.**
> 누가 왔는지, 몇 명인지 확정하지 않으면 타임라인 전체가 오염된다.

```
[검증 항목 A] 주어 단수/복수 확인
  질문: "혼자(alone)" 단어가 명시되어 있는가?
  → YES: 단독 행동 확정
  → NO:  단독으로 가정하지 마라 — 다음 절의 대명사를 반드시 확인

[검증 항목 B] 1인칭 복수 트리거 — "WE/we/우리" 감지
  규칙: 단수 주어(A가 왔다) → 그 입에서 "우리" 발화 → 동행자 존재 확정
  예시: 요 20:1 "cometh Mary Magdalene" (단수 서술)
        요 20:2 "WE know not where they have laid him" (복수!)
        → 마리아(막달라)는 혼자가 아니었다 ← TYPE-M 즉시 발동

[검증 항목 C] 대명사 격 일치 확인
  질문: 본문에 등장하는 모든 대명사(he/she/they/we)가
        앞서 확정된 인물 수와 일치하는가?
  → 불일치 발견 시: TYPE-M 발동 → 숨겨진 인물 추적

[검증 항목 D] 천사/인물 묘사 충돌 판별
  같은 방문에서 묘사가 다르면:
  → 합리적 변화(시간 경과, 시점 차이)로 설명 가능한가?
     YES: TYPE-C 적용 (출현 동작 vs 정적 상태)
     NO:  다른 방문으로 분리 처리 (DE-OVERLAP 발동)
  묘사 충돌 허용 기준:
     ✅ 허용: 나타날 때 서 있었다 → 이후 앉아있다 (합리적 변화)
     ❌ 불허: 외부에 앉아 있다 + 내부에 앉아 있다 → 동시 불가 → 다른 방문

[검증 항목 E] 입장 동사 검증 (Entry Verb Verification) ← 오늘 추가된 항목
  규칙: 각 인물이 공간(무덤/방/집 등)에 "실제로 들어갔는가"를
        반드시 원문 동사로 확인한다.
  판별:
    → 원문에 "went in" / "entered" / "came into" 등 입장 동사 있음 → 들어간 것 확정
    → 원문에 입장 동사 없음 → 들어가지 않은 것으로 처리 (가정 금지)
  실패 사례:
    요 20:1 "seeth the stone taken away" / "runneth" → 입장 동사 없음
    → 마리아(막달라)는 무덤에 들어가지 않음 확정
    → 이를 "들어갔다"고 기술한 것은 원문 근거 없는 오류 ❌
    → 입장하지 않았으므로 천사 메시지도 못 들음 → "They have taken away" = 도난 의심 발화

[검증 항목 F] 발화자 단수/복수 검증 (Speaker Plurality Check) ← 오늘 추가된 항목
  규칙: 인용된 발화의 주어가 단수인지 복수인지 반드시 원문으로 확인한다.
  판별:
    → "they said" / "they spake" → 복수 발화자
    → "he said" / "she said" / "saith" → 단수 발화자
  실패 방지:
    눅 24:5 "they said unto them" = 두 천사가 함께 발화 → 복수 기록
    막 16:6 "he saith unto them"   = 1명이 발화 → 단수 기록
    → TYPE-C 적용: 누가 = 전체 발화자 기록 / 마가 = 대표 발화자 기록
    → 혼동 없이 기술하려면 반드시 원문 주어 동사 확인 선행

[검증 항목 G] 인물별 동선·접근 정보 추적표 (Character Movement & Information Access Map)
  ← 이것이 없으면 "왜 A는 알고 B는 모르는가"를 설명할 수 없다.

  규칙: 장면에 등장하는 각 인물에 대해 아래 표를 작성한다.
  ┌─────────────────────────────────────────────────────────┐
  │ 인물 │ 출발 → 이동경로 → 위치 │ 볼 수 있는 것 │ 들을 수 있는 것 │ 결과 정보 │
  └─────────────────────────────────────────────────────────┘

  활용 예시 (부활 아침):
  | 인물 | 이동경로 | 볼 수 있는 것 | 들을 수 있는 것 | 결과 정보 |
  |:---|:---|:---|:---|:---|
  | 마리아(막달라) | 무덤 앞까지 → 달려감 | 돌 굴려짐 | 천사 메시지 없음(입장 안 함) | "They have taken away" (도난 의심) |
  | 여인들 그룹 | 무덤 안까지 들어감 | 천사 2명 | "He is risen!" 들음 | 부활 메시지 수신 |
  | 베드로 | 무덤 안까지 들어감 | 세마포·수건 | — | 시신 없음 확인 |

  이 표 없이 "왜 막달라는 부활을 몰랐는가"를 설명하면 → 근거 없는 주장이 됨.

  묘사 차이 판별 3가지 유형:
  ┌─ [상태 변경 State Change]:
  │   같은 대상, 시간 경과 → 상태가 바뀜
  │   예: 천사가 나타날 때 서 있었다 → 이후 앉아서 말함
  │   → TYPE-C 적용 (합리적 변화 ✅)
  │
  ├─ [환경 변경 Environment Change]:
  │   다른 방문자 / 다른 시간 / 다른 장소 → 다른 결과
  │   예: T1 천사 내부 서 있음 / T4 천사 내부 앉아 있음 → 다른 방문 시각
  │   → DE-OVERLAP 적용 (다른 방문으로 분리)
  │
  └─ [시점 차이 Perspective Difference]:
      같은 순간, 다른 관찰자 / 다른 기록 초점
      예: 눅 = 전체 발화자(둘 다) / 막 = 대표 발화자(1명)
      → TYPE-C 적용 (기록 초점 차이 ✅)

  판별 순서:
  1. 먼저 "이동경로표" 작성 → 각 인물의 접근 가능 정보 확정
  2. 묘사 차이 발견 시 → 상태변경 / 환경변경 / 시점차이 중 하나로 분류
  3. 분류 근거를 KJV 원문 동사·명사로 확인
  4. 분류 불가 시 → TYPE-M 발동
```

**⚠️ 이 PRE-STEP 0을 건너뛰고 TYPE-B를 발동하는 것은 금지된다.**
**PRE-STEP 0 통과 전까지 타임라인 구성 시작 불가.**

**📌 실패 사례 누적 교훈:**
```
오류 1: 요 20:1 "cometh Mary" → "혼자" 가정 → 항목 A,B 위반
오류 2: 막달라가 무덤에 들어갔다고 기술 → 항목 E 위반 (입장 동사 없음)
오류 3: "서/앉" 동시 표기 → 항목 D 위반 (자기모순)
오류 4: 발화자 단복수 미확인 → 항목 F 위반
→ 4개 오류 모두 PRE-STEP 0 항목이 먼저 있었다면 방지 가능했음
```

## 실행 단계

```
[PRE-STEP 1] 원어 파싱 명세서 (Original Parsing Spec) 출력 의무화
  → KJV 영어 뉘앙스에 끌려가는 것을 막기 위해, 핵심 헬라어/히브리어 단어의 문법표를 논리 전개 전에 가장 먼저 출력한다.
  → 형식:
    * 단어: [원어] (발음)
    * 품사: [예: 관계 부사, 접속사 등]
    * 성/수/격: [예: 남성/단수/주격 등 (해당 시)]
    * 사전적 기능: [예: 물리적 장소 지시, 단순 연결이 아닌 확장 등]
    * 비교 단어와의 차이: [경쟁 해석에 쓰이는 단어와 문법적 차이점]

[STEP 1] 공격받는 구절의 KJV 원문 전체 인용
  → 삭제 주장 구절 앞뒤 2~3절을 포함하여 KJV 원문을 완전히 인용.

[STEP 2] 접속사/콤마 색별 그룹핑 분석
  → KJV의 and/comma/the 등을 색으로 구분하여 그룹을 시각화.
  → | 그룹 | 포함 단어들 | 접속사 구조 | 의미 |
  → 평면적 나열인지, 의도된 하위 그룹핑인지 판별.

[STEP 3] 삭제 시 문법/신학 체인 단절 확인 (핵심)
  → 공격받는 구절을 삭제했을 때 어떤 문법적 이상이 발생하는지 확인.
  → 남성 분사가 중성 명사를 수식하는 성(Gender) 불일치 등 원어 문법 오류를 도출.
  → 예: 요일 5:7 삭제 시 → 8절의 남성 분사(οἱ μαρτυροῦντες)가
         중성 명사들을 받아야 하는 문법 붕괴 발생.

[STEP 4] 하늘/땅 대칭 구조 검증
  → "삭제 시 대칭 구조가 붕괴되는가?"를 확인.
  → 7절(하늘)과 8절(땅)이 반드시 세트로 있거나 없는 패턴 → 구조적 설계 증명.

[STEP 5] 다른 구절에서의 동일 패턴 교차 검증
  → 동일한 접속사 그룹핑 패턴이 성경 다른 곳에서도 사용되는지 확인.
```

## 📌 [서브 프로토콜] 이스마엘 제7규칙 (상호 한정 구문 검증)

> **"כלל שהוא צריך לפרט ופרט שהוא צריך לכלל"** (Kelal she-hu zerik li-ferat...)
> (구체적 사례의 설명을 요구하는 일반 원칙, 일반 원칙의 설명을 요구하는 구체적 사례)

```
[발동 조건]
  → 적이 동일 구절 내의 두 구문을 가리켜 "불필요한 동어 반복(Redundancy)이다" 혹은 "서로 충돌한다"고 공격할 때.

[실행 로직]
  1. 두 구문 중 어느 것이 **일반 원칙(Kelal)**이고 어느 것이 **구체적 사례(Perat)**인지 식별한다.
  2. 일반 원칙만 있을 때 발생하는 오독(오류 A)을 증명한다.
  3. 구체적 사례만 있을 때 발생하는 오독(오류 B)을 증명한다.
  4. 두 구문이 결합하여 상호 한정(Mutually Limiting)함으로써 오류 A와 B를 모두 완벽하게 방어하는 정밀한 설계임을 선언한다.

[레퍼런스 케이스: 출애굽기 13:2]
  KJV 원문: "Sanctify unto me all the firstborn (일반), whatsoever openeth the womb (구체)..."
  오독 A (일반 원칙만 있을 때): "모든 처음 난 자" → 제왕절개로 태어난 첫째도 포함될 위험.
  오독 B (구체적 사례만 있을 때): "태를 여는 것" → 이전 유산 후 태를 열고 나온 '둘째'도 포함될 위험.
  결론: 두 구문은 동어 반복이 아니라 서로의 약점을 완벽히 막아주는 '상호 의존적 정밀 설계'이다.
```

## 📌 레퍼런스 케이스 1: 요한1서 5:7 콤마 구절

```
[KJV 원문]:
  요일 5:7: "For there are three that bear record in heaven,
             the Father, the Word, and the Holy Ghost: and these three are one."
  요일 5:8: "And there are three that bear witness in earth,
             the spirit, and the water, and the blood: and these three agree in one."

[삭제 시 문법 붕괴]:
  8절: "οἱ μαρτυροῦντες" (남성 복수 분사)
  → "τὸ πνεῦμα καὶ τὸ ὕδωρ καὶ τὸ αἷμα" (중성 명사들)를 수식
  → 7절 없으면: 남성 분사가 중성 명사 수식 → 헬라어 성(Gender) 불일치 문법 오류 ❌
  → 7절 있으면: 남성 명사들(ὁ Πατήρ, ὁ Λόγος)이 선행사 → 문법 완벽 ✅

[하늘/땅 대칭]:
  7절: ἐν τῷ οὐρανῷ (하늘에서) — 성부, 말씀, 성령
  8절: ἐν τῇ γῇ (땅에서) — 성령, 물, 피
  → 하늘 증인 + 땅 증인의 이중 증언 구조 → 7절 삭제 시 대칭 붕괴 ✅
```

## 📌 레퍼런스 케이스 2: 역대기상 1장 팀나 족보

```
[KJV 원문]:
  "The sons of Eliphaz; Teman, AND Omar, Zephi, AND Gatam, Kenaz, AND Timna, AND Amalek."

[접속사 그룹핑 분석]:
  그룹 1 (노란색): Teman + Omar — and로 쌍을 이룸 (정실 아들 쌍)
  그룹 2 (초록색): Zephi + Gatam — and로 쌍을 이룸 (정실 아들 쌍)
  그룹 3 (빨간색): Kenaz + Timna + Amalek — 한 단위로 묶임
                   = 첩 팀나(창 36:12)와 그녀의 아들 아말렉을 포함한 한 가문 단위

[결론]:
  무작위 나열이었다면 이 정교한 쌍 구조가 불가능.
  → 팀나는 딸이 아니라 첩(concubine)이라는 창 36:12의 기록과 완벽히 정합 ✅
  → KJV의 and/comma 구조가 족보 정보를 압축하여 담고 있음을 증명 ✅
```

## 🆕 서브 프로토콜: 동사 차이로 목적지 분리 (Verb Differentiation) — v1.1

> **발동 조건**: 적이 "같은 표현이므로 같은 목적지"라고 주장할 때.
> **핵심 질문**: *"같은 저자가 같은 목적지를 지시할 때, 정말 같은 동사를 쓰는가?"*

```
[STEP 1] 공격 대상 구절과 비교 구절의 핵심 동사를 원어로 분리한다
  → 예: 요 13:1 μεταβαίνω vs 요 13:33/36 ὑπάγω

[STEP 2] 같은 저자가 같은 목적지에 대해 쓴 동사를 전수 검색한다
  → 목적지 명시 여부 확인: 동사 + "아버지" 명시 vs 동사 + 목적지 미명시

[STEP 3] 목적지 명시 패턴이 일관적이면 → 미명시 = 다른 목적지로 판정
  → 전례: 요 14-16에서 ὑπάγω+아버지 반복 명시 → 13:33/36만 미명시
    → 의도적 침묵 = 다른 측면(십자가 경로)을 가리킴
```

**전례**: `[F+E+G+N+P+I+T+S]_베드로_갈보리순교설.md` — TYPE-G 최강 방어 3단계

## 🆕 서브 프로토콜: 부사→명사 락인 (Adverb-Noun Lock-in) — v1.1

> **발동 조건**: 장소 관계부사(ὅπου 등)가 목적지를 암시하나 명시하지 않을 때.
> **핵심 질문**: *"이 부사가 가리키는 실체적 명사(τόπος)가 같은 저자의 텍스트 내에 존재하는가?"*

```
[STEP 1] 장소 관계부사(ὅπου)가 사용된 예언/선언 구절을 식별한다
  → 예: 요 13:36 "ὅπου ὑπάγω" — 목적지 미명시

[STEP 2] 같은 저자의 텍스트에서 부사의 명사적 해소(Nominal Resolution)를 검색

[STEP 3] 부사가 명사를 찾으면 → 예언의 성취 장소 확정
  → "부사가 명사를 찾았다. 예언이 성취 장소를 찾았다."
```

## 🆕 서브 프로토콜: 묶음 대명사 이중 목적지 지시 불가 (Bound Pronoun Dual-Destination Impossibility) — v1.1

> **발동 조건**: 적이 하나의 위치 대명사(עִמִּי, μετ᾽ ἐμοῦ 등)로 묶인 복수 대상이 **서로 다른 물리적 장소**에 있을 수 있다고 주장할 때.
> **핵심 질문**: *"하나의 '나와 함께'로 묶인 대상들이 물리적으로 건너갈 수 없는 두 공간에 동시에 존재할 수 있는가?"*

```
[STEP 1] 묶음 대명사 식별
  → 한 문장에서 복수의 대상을 하나의 위치 대명사로 묶는 구조를 식별한다.
  → 예: 삼상 28:19 "너(사울)와 네 아들들(요나단)이 나와 함께(עִמִּי) 있으리라"
  → 핵심: עִמִּי(immi) = "나와 함께" — 단일 위치 지시

[STEP 2] 공간 구조 확인 (물리적 경계 존재 여부)
  → 해당 위치에 물리적으로 건너갈 수 없는 경계가 성경에 명시되어 있는가?
  → 예: 눅 16:26 "큰 구렁(great gulf)이 고정되어 있어 건너갈 수 없다"
  → 경계 존재 확인 → STEP 3 진행

[STEP 3] 이중 지시 모순 검증
  → 적의 주장대로 묶인 대상들이 경계의 양쪽에 분리 배치된다면:
    → "나와 함께(עִמִּי)" = 낙원(사무엘 위치) + 고통의 장소
    → 하나의 대명사가 건너갈 수 없는 두 공간을 동시에 지시하는 문법적 괴물 발생
  → 이중 지시 모순 확정 ❌

[STEP 4] 판정
  → 묶음 대명사로 묶인 이상, 모든 대상은 반드시 동일 위치에 있어야 함
  → "나와 함께"는 포괄적 관용구가 아니라, 구렁 없는 동일 위치를 가리키는 공간 특정 선언
```

## 🆕 서브 프로토콜: 구조적 쌍 잠금 (Structural Pair Lock) — v1.2

> **발동 조건**: 상대방이 텍스트의 일부 구절(절)을 삭제하거나 후대 삽입이라 주장할 때.
> **핵심 질문**: *"삭제 대상 구절이 없으면, 남아있는 구절의 문법·구조·어휘가 스스로 붕괴되는가?"*
> **핵심 원리**: 원본 설계에서 두 요소가 구조적 쌍(Structural Pair)으로 설계되었다면, 하나를 제거할 때 나머지에 반드시 고아 어휘(Orphan Word)나 문법 공백이 발생한다.

```
[STEP 1] 문제 구절과 인접 구절의 대칭 구조 도식화
  → 의심 구절(삭제 주장 대상)과 나머지 구절을 나란히 놓는다.
  → 공간적 대조어, 반복 어휘, 문법적 연결어를 표시한다.
  → 표 형식: | 구절 | 핵심 어휘/구조 | 대응 구절 | 대칭 관계 |

[STEP 2] 삭제 시뮬레이션
  → 의심 구절을 가상으로 제거한다.
  → 남은 본문에서 다음 현상이 발생하는지 확인:
     □ 고아 어휘(Orphan Word): 앞뒤 맥락 없이 홀로 남는 단어
     □ 문법 공백: 연결 접속사나 관계대명사의 연결 대상이 사라짐
     □ 대칭 붕괴: 구조적으로 대응하던 쌍이 깨져 의미 불균형 발생

[STEP 3] 전 사본 계열에서 쌍 일관성 확인
  → Comma를 포함하는 모든 사본 계열에서 두 구조 요소가
    반드시 '세트로 있거나 세트로 없는지' 확인한다.
  → 세트 패턴이 확인되면 = 원본 설계 증명

[STEP 4] 판정
  → 삭제 시 고아 어휘/문법 공백 발생 + 전 계열 세트 패턴
  → "이 구절은 삭제가 아니라 나머지 구절의 문법적 근거다" 선언
```

## 🆕 서브 프로토콜: 헬라어 성(Gender) 문법 충돌 적발 (Greek Gender Mismatch Detection) — v1.3

> **발동 조건**: 상대방이 헬라어 구절의 일부를 삭제하거나 재해석할 때, 삭제 후 남은 구절에서 헬라어 문법의 성(性) 불일치가 발생하는지 확인한다.
> **핵심 원리**: 헬라어는 형용사·분사·관사의 성이 수식하는 명사와 일치해야 한다. 삭제 후 성 불일치가 발생하면 삭제가 원본 구조를 파괴했다는 문법적 증거가 된다.

```
[STEP 1] 삭제 후 본문에서 분사/관사/대명사의 성·수 추출
  → 의심 구절을 가상으로 제거한다.
  → 남은 본문에서 성(性)이 있는 주요 문법 요소를 목록화:
    □ 분사(Participle): 남성(ὁ)/여성(ἡ)/중성(τό) 어미 확인
    □ 관사(Article): 남성 복수(οἱ)/여성 복수(αἱ)/중성 복수(τά) 확인
    □ 지시 대명사: 가리키는 명사의 성과 일치 여부 확인

[STEP 2] 성 일치 대상 명사 확인
  → 분사·관사가 가리키는 명사들의 성을 확인한다.
  → 불일치 발생 여부:
    □ 일치: 삭제 후에도 문법 정합 → 삭제 주장에 유리
    □ 불일치: 삭제로 인해 문법 오류 발생 → 원본 구절의 필요성 증명

[STEP 3] 불일치의 신학적 의미 해석
  → 성 불일치가 발생한 이유는 무엇인가?
  → 삭제된 구절 안에 있었던 요소(남성 인격체 등)가
    남은 구절의 분사·관사를 지탱하고 있었음을 증명한다.

[STEP 4] 판정
  → 성 불일치 발생 확인 + 삭제된 구절로만 해소 가능
  → "이 구절은 삭제될 수 없다. 인접 구절의 헬라어 문법이 이 구절의 존재를 요구한다." 선언
```

### 📌 레퍼런스 케이스: 요한1서 5:7-8 — 남성 복수 분사 vs 중성 명사 충돌

```
[STEP 1] 삭제 후 본문 분석 (NA28 — 7절 삭제된 상태):

    오직 중성 명사 셋만 남음 → 성 불일치 ❌

[STEP 4] 판정]:
  "7절(콤마)이 없으면 8절의 남성 복수 분사
   οἱ μαρτυροῦντες가 중성 명사 셋을 가리키게 되어
   헬라어 성(gender) 법칙을 위반한다.

   본문 내적 증거다."
```

### 🔴 우선 개발 대상 (미비 3종)

**① Granville Sharp Rule (G-CHK-4)**
> 헬라어 구조: 정관사(ὁ) + 명사1 + καί + 명사2
> → 관사가 하나일 때 두 명사는 동일 인물 지시
> 예: "τοῦ θεοῦ ἡμῶν καὶ σωτῆρος Ἰησοῦ Χριστοῦ" (딛 2:13)
> → "우리 하나님 **곧** 구주 예수 그리스도" = 하나님 = 예수 (삼위일체 증거)

**② 히브리어 동사 어간 Binyan (G-CHK-6)**
> Qal = 단순 능동 / Niphal = 수동 또는 재귀 / Piel = 강조·반복
> Hiphil = 사역(~하게 하다) / Hithpael = 재귀(스스로 ~하다)
> → 같은 어근도 어간이 다르면 의미가 완전히 달라짐

**③ 선지자적 완료형 (G-CHK-7)**
> 히브리어 완료형 시제가 미래 예언에 쓰이는 경우
> → "그가 찔렸다(עֲנֹה)" (사 53:5) — 이사야 당시엔 미래 사건
> → 적이 "완료형 = 과거 역사 사건"으로 오독할 때 역공 가능
