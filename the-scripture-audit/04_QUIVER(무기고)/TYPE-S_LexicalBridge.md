# 🏹 TYPE-S: 어휘 교차 연결 (Shared Lexical Bridge)

> **별칭**: 게제라 샤바(גְּזֵרָה שָׁוָה, Gezerah Shavah)
> **히브리 전통**: 힐렐의 7대 해석 규칙 중 제2규칙 — "동일 단어, 동일 원칙"
> **핵심 철학**: 서로 다른 두 구절에서 동일한 핵심 단어가 사용되었다면, 한 구절에서 확정된 의미가 다른 구절에도 적용된다.

> [!IMPORTANT]
> TYPE-S는 TYPE-I(빈도 카운팅)과 다르다.
> TYPE-I = 단어가 **몇 번** 나오는가 (숫자 대칭/설계)
> TYPE-S = 단어가 **어디에 또** 나오는가 (의미적 교차 연결)
> TYPE-I는 빈도를 세고, TYPE-S는 의미를 잇는다.

---

## 발동 조건

```
다음 상황에서 TYPE-S를 발동한다:
1. 해석이 불확실한 구절에서 핵심 단어/어근이 발견됨
2. 그 동일한 단어가 다른 구절에서 확정된 의미로 사용됨
3. 두 구절의 어휘 연결이 신학적 브리지를 형성할 가능성이 있음
```

---

## 실행 단계

```
[STEP 1] 핵심 어휘 식별 (Keyword Isolation)
  → 해석 대상 구절에서 핵심 동사/명사/형용사를 원어(히브리어/헬라어)로 분리한다.
  → 번역본이 아닌 원어 어근(root)을 기준으로 식별한다.

[STEP 2] 교차 검색 (Cross-Reference Search)
  → 동일한 원어 단어가 성경 전체에서 사용된 다른 구절들을 수집한다.
  → 특히 동일 저자, 동일 문맥, 동일 인물 관련 구절을 우선 검색한다.

[STEP 3] 의미 확정 구절 선별 (Anchor Verse Selection)
  → 수집된 구절들 중 해당 단어의 의미가 문맥적으로 확정된 구절을 앵커로 선정한다.
  → 앵커 구절에서의 의미는 논쟁의 여지가 없어야 한다.

[STEP 4] 브리지 구축 (Bridge Construction)
  → 앵커 구절의 확정된 의미를 해석 대상 구절에 적용한다.
  → 적용 결과가 문맥적으로 일관성이 있는지 검증한다.

[STEP 5] 반격 봉쇄 (Counter-Attack Blocking)
  → "단순한 동일 어휘일 뿐 의미 연결은 아니다"라는 반론에 대해:
     a) 동일 저자인가? (저자 의도 일치 가능성 ↑)
     b) 동일 인물/사건 관련인가? (문맥 연결 ↑)
     c) 다른 TYPE(N/P/G 등)이 이 연결을 독립적으로 지지하는가? (다중 검증)
```

## 📌 레퍼런스 케이스: θέλω 어휘 연결 — 베드로 갈보리 순교 확정

```
[해석 대상] 요 21:18 — "원치 않는 곳(ὅπου οὐ θέλεις)으로 너를 옮겨 가리라"
  → 핵심 어휘: θέλω (원하다) + ὅπου (장소)
  → 질문: 베드로가 원치 않지만 옮겨가는 "곳"은 어디인가?

[STEP 1] 핵심 어휘 = θέλω (thélō, 원하다)

[STEP 2] 교차 검색 결과:
  → 마 26:39 — "not as I will(θέλω), but as thou wilt"
     예수님이 겟세마네에서 "원치 않으셨으나" 순종하신 장면

[STEP 3] 앵커 구절 = 마 26:39
  → 예수님이 θέλω로 원치 않았으나 간 곳 = 갈보리/골고다 (요 19:17)
  → 이 의미는 논쟁의 여지 없이 확정됨 ✅

[STEP 4] 브리지 구축:
  → 예수님: θέλω (원치 않음) → 갈보리
  → 베드로: θέλω (원치 않음) → ??? 
  → + "Follow me" (요 21:19) = 같은 경로
  → ??? = 갈보리 ✅

[STEP 5] 반격 봉쇄:
  a) 동일 저자? → ✅ 요한이 마태의 겟세마네 기록을 알고 있었음 (겟세마네에 함께 있었음)
  b) 동일 인물? → ✅ 예수님-베드로 관계, 동일 사건 체인 (겟세마네→갈보리→부활→요21)
  c) 다른 TYPE 지지? → ✅ TYPE-N(배타성), TYPE-P(역논법), TYPE-G(ὅπου 장소 부사)
  → 다중 검증 통과 → CONFIRMED ✅
```

## 💡 TYPE-S의 핵심 가치

```
기존 학계의 방식:    구절 A를 독립적으로 해석 → 여러 가능성 중 하나 선택
TYPE-S의 방식:      구절 A의 핵심 단어가 구절 B에서도 사용됨을 발견
                     → 구절 B에서 그 단어의 의미가 확정됨
                     → 확정된 의미를 구절 A에 적용
                     → 해석의 가능성이 하나로 수렴
                     = 어휘 브리지로 불확실성을 제거
```

## ⚠️ TYPE-S 사용 시 주의사항

> [!WARNING]
> 아래의 경우 TYPE-S 적용을 자제해야 한다.

| ❌ 위험한 적용 | ✅ 안전한 적용 |
|:---|:---|
| 히브리어/헬라어 어근이 다른데 번역본에서만 같은 단어 | 원어 어근(root)이 동일한 경우에만 적용 |
| 동일 단어이나 문법적 기능이 완전히 다름 (동사↔명사) | 동일 품사, 유사 문법 구조에서의 사용 |
| 연결을 지지하는 다른 TYPE이 전무 | TYPE-N/P/G 등 최소 1개 이상의 독립 검증 |
| 앵커 구절의 의미 자체가 논쟁적 | 앵커 구절의 의미가 문맥적으로 확정됨 |
