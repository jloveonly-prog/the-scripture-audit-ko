# 🔗 ANCHOR — 제3 앵커 수집 + MATRIX 역산 프로토콜
**"In the mouth of two or three witnesses every word shall be established." — 2 Corinthians 13:1 KJV**
**"For precept must be upon precept... here a little, and there a little." — Isaiah 28:10 KJV**

> **핵심 원칙**: 충돌하는 두 구절만 비교하면 영원히 답이 나오지 않는다.
> 반드시 제3의 독립 데이터 포인트를 찾아야 퍼즐이 완성된다.

## ⚙️ 실행 절차

```
STEP 1. 충돌 구절 목록화
  구절A: [KJV 인용] — 제시하는 값: [숫자/사실]
  구절B: [KJV 인용] — 제시하는 값: [숫자/사실]
  충돌 지점: [구체적으로 무엇이 다른가]

STEP 2. 제3 앵커 검색 (필수 — 절대 생략 금지)
  → 동일 인물/사건과 관련된 성경의 모든 구절에서 숨겨진 수치를 수집한다.
  → [앵커 강제 확장 검색 알고리즘 3단계]
    * 1차 검색 (근접 문맥): 충돌 구절의 앞뒤 5장 이내에서 나이, 연수, 기간 명시 구절 스캔
    * 2차 검색 (병행 본문 전수조사): 열왕기 ↔ 역대기, 복음서 간 동일 사건 기록 크로스체크
    * 3차 검색 (계보/생물학적 역추적): 아버지의 즉위 나이, 아들의 통치 기간 등 족보 데이터 조회
  → 🚨 비상 탈출: 위 3단계 후에도 수치가 나오지 않으면 "ANCHOR 발견 실패: 추가 데이터 필요"를 선언하고 사용자에게 힌트를 요청한 뒤 분석을 강제 중단한다 (소설 작성 금지).
  → 수집된 데이터 표 형식:
    | 구절 | 데이터 유형 | 값 | 구절A·B와의 연관성 |

STEP 3. 앵커 유효성 검증
  → 제3 앵커가 구절A 또는 B와 독립적인가?
    (같은 사건을 다른 책/저자/각도로 기록)
  → 앵커의 숫자/사실이 A+B의 합산 또는 역산에 사용 가능한가?
  → NO → 추가 앵커 검색 반복

STEP 4. 앵커 대입 시험
  → [앵커값] ± [구절A값] = [구절B값] 이 성립하는가?
  → 성립 → STEP 5 (MATRIX 역산)으로 진행
  → 불성립 → STEP 2로 돌아가 다른 앵커 검색
```

## 📐 STEP 5. MATRIX 역산 (수학적 검증)

> **이 단계가 없으면 증명이 아니다 — 반드시 수식으로 완결한다.**

```
MATRIX 역산 3원칙:

원칙 1. 수식화
  → 모든 수치를 하나의 수식으로 표현한다
  → 형식: [앵커A] + [앵커B] + ... = [구절X의 값]
  → 예: 32(즉위) + 8(통치) + 2(투병) = 42 ✅

원칙 2. 양방향 검증
  → 순방향: A + B = C 성립?
  → 역방향: C - B = A 성립?
  → 양방향 모두 성립해야 확정

원칙 3. 잉여 데이터 확인
  → 수식에 포함되지 않은 성경 수치가 있으면
    그 수치도 수식과 모순이 없는지 확인
  → 모순 없음 → 판결 확정
  → 모순 있음 → TYPE-M 재발동 (새 이상 신호 포착)
```

**MATRIX 출력 형식 (표로 반드시 시각화):**

| 구절 | 데이터 | 값 | 수식 내 역할 |
|:---|:---|:---:|:---|
| 대하 21:5 | 여호람 즉위 나이 | 32 | 기준점 |
| 대하 21:20 | 여호람 통치 기간 | 8 | 누적 A |
| 대하 21:18 | 여호람 투병 기간 | 2 | 누적 B |
| **합계** | **= 아하지야 탄생 시 부친 나이** | **42** | ✅ 대하 22:2 해소 |

## 🧑‍⚖️ 교차증인 독립성 검증 (Cross-Witness Independence Check) — v1.1 신설

> **발동 조건**: COMBO-VERIFY에서 TYPE 3종 이상이 수렴하여 IRONCLAD 후보가 확정된 직후, STRESS-TEST-7과는 별개로 반드시 실행한다.
> **핵심 질문**: *"수렴한 TYPE 무기들이 실제로 서로 다른 책(저자)에서 나온 독립 구절에 적용되었는가, 아니면 같은 구절 1~2개를 여러 각도로 분석한 것뿐인가?"*
> **성경적 근거**: 신 19:15, 마 18:16, 고후 13:1 — "두세 증인의 입으로 말마다 확정하라." 증인은 서로 다른 사람이어야 증언으로 성립하듯, 성경 해석의 "증인"도 서로 다른 책/저자에서 나온 독립 구절이어야 한다. TYPE 무기 3종이 전부 같은 구절에 적용된 것은 "증인 1명이 세 가지 방식으로 증언한 것"이지 "증인 3명의 증언"이 아니다.

### 실행 절차

```
[STEP 1] 수렴 TYPE별 근거 구절 목록화
  → COMBO-VERIFY에서 발동한 각 TYPE이 어느 구절에 적용되었는지 표로 정리한다.
  → | TYPE | 적용 구절 | 소속 책(저자) |

[STEP 2] 책(저자) 독립성 카운트
  → 위 표에서 서로 다른 책(저자)이 몇 개 등장하는지 센다.
  → 같은 책의 인접 구절(예: 창 3:16과 창 3:20)은 독립 증인 1개로만 카운트한다.
  → 완전히 다른 책/저자(예: 욥기 + 히브리서 + 유다서)는 각각 독립 증인으로 카운트한다.

[STEP 3] 태그 부여
  → 서로 다른 책(저자) ≥ 2개 확인 → ✅ [교차증인 ✓ — N개 독립 책/저자]
  → 서로 다른 책(저자) ≤ 1개(동일 구절/동일 저자 내부 반복) → ⚠️ [교차증인 미확인]

[STEP 4] 신학적 결정 표로 전달
  → 이 태그는 `BVCAP_GHQ.md` PHASE 5 "7. 신학적 결정" 표의 입력값으로 쓰인다.
  → [교차증인 ✓] + IRONCLAD → 확정(Doctrine)으로 표기 가능
  → [교차증인 미확인] + IRONCLAD → 내부 등급(IRONCLAD)은 그대로 유지하되, 목회 적용 등급은 개연성(Probability)으로 하향 표기 — 교리화 금지
```

## 📌 레퍼런스 케이스

```
[아하지야 난제]
  구절A: 대하 22:2 — 아하지야 즉위 시 42세
  구절B: 왕하 8:26 — 아하지야 즉위 시 22세
  충돌: 42세 vs 22세

  앵커 검색:
  → 대하 21:5  — 여호람 즉위 시 32세
  → 대하 21:20 — 여호람 8년 통치
  → 대하 21:18 — 여호람 2년 투병 후 사망  ← 제3 앵커 확보

  MATRIX 역산:
  32 + 8 + 2 = 42 ← 아하지야 탄생 시 부친 여호람의 나이
  → 22세(왕하) = 아하지야 자신의 나이
  → 42세(대하) = 왕조 기준 부친 나이 좌표
  → 모순 없음 ✅

[솔로몬 외양간 난제]
  구절A: 왕상 4:26 — 외양간 40,000칸
  구절B: 대하 9:25 — 외양간 4,000칸
  충돌: 40,000 vs 4,000

  앵커 검색:
  → 왕상 10:26 — 병거 1,400대  ← 제3 앵커 확보

  MATRIX 역산:
  1,400(병거) × 3(마리/대) = 4,200 ≈ 4,000(작전 단위) ✅
  4,000(작전 단위) × 10(개별 수용칸) = 40,000(전체 수용칸) ✅
  → 4,000 = 병거 운용 작전 단위 외양간
  → 40,000 = 개별 말 수용 칸막이 총수
  → 동일한 시설을 다른 기능 기준으로 계산한 것 ✅
```
